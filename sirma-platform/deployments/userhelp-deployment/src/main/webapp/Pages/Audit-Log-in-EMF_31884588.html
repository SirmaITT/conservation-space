Sirma Platform : Audit Log in EMF

In the Audit log are recorded all activities of all users and an user with administrative permissions is able to track these activities.

Actions recorded in the Audit log

A record in the audit log is created when one of the following events occurs:

  1. Login
  2. Logout, session expiration/ termination
  3. All actions on all kinds of domain objects (project, case, object, document, comment, saved search etc.)
  4. Automatic system actions
  5. Search - only the type (basic, keyword, advanced)

New entries are not recorded in the audit log when the user performs:

  1. Search in pop-up
  2. Navigation
  3. Actions with widgets and formatting in iDoc/Object in edit mode
  4. Formatting in IAT
  5. Zoom in/ Zoom out
  6. Expand/collapse sections (iDoc), navigation tree
  7. Download document
  8. Open object for preview

Data collected in the Audit log

The following information is stored in the audit log and the user is able to select it as a separated search criteria:

  1. Action performed
  2. Date and time when action was performed
  3. User who performed the action
  4. Object type
  5. Object subtype
  6. Object primary state
  7. Object secondary state
  8. Object’s previous state
  9. Object title + link to the object (to see the object's details)
  10. Object ID – the business identifier
  11. Object system ID – the system unique identifier of the object (usually not visible to the user)
  12. Context - stores the context of the object on which the action is performed. For the first version only Document library, Object library, Project and Case context will be handled.
  13. Relation to - when Add relation action is logged, this value shows the object details to which the object is linked (related).
