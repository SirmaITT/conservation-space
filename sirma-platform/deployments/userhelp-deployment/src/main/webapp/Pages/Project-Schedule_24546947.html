Sirma Platform : Project Schedule

Manage project schedule functionality described in this section is intended to help the project managers and team leads in their day to day activities in project planning, creation of WBS, assigning resources to tasks, detecting resources work load and availability on projects, and tracking project progress and performance.
The main advantage comes from the fact that the project timeline is kept constantly up to date with the real project work by automatic updates that happen with each user task completion or creation of a new task in the project and sub project work spaces.

In the project schedule functionality each of the following primary system objects is treated and represented as a task:

  • Project - only the task of basic type "Project" could not be created directly into the project schedule. One project schedule could have only one task of type"Project".
  • Sub-project/ case - Sub-projects/ cases could be created both from the project working area, or from the project schedule, as tasks of basic type "Sub-project".
  • Workflow - Workflows could be started and created again both from the project working area, or from the project schedule, as tasks of basic type "Workflow".
  • Task - each task being a summary task, or a leaf task, part of workflow or not, is treated as a task of basic type "Task".

Basic task types in the Project Schedule are:

  • Project
  • Sub-project
  • Workflow
  • Task
  • Milestone - milestones are normal tasks but with no effort (TBD: and assignee)

These basic task types actually represents the object instances that will be created from the project schedule. Because each primary object itself has its own specific type (like Project type, Workflow type, Sub-project type, Task type) then based on the primary task type the user will be able to select the specific type.

For example a task with basic type "Workflow" will have specific types "Review and Approval", "Test new part", "Design", etc.
