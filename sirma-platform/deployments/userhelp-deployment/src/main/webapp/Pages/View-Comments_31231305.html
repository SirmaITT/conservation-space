Sirma Platform : View Comments

Each user who has permissions to an object can view the comments on it.

  • In each object landing page on the right side there is a section "Comments", where the user could view all comments on the object.

'Comment thread' we call a main comment (the first from the list -  with the title) and all the replies on it. Only the first comment in the thread has a title, the replies do not. There could be more than one comment thread for an object.

    • The comment threads in the section are in the view of accordion.
    • The comment threads are collapsed - we see only the main comment and not the replies.

    • For each comment thread we see:
      • Picture of user who started the comment
      • Name of the user
      • Title of the comment
      • Date of creation
    • The latest threads are on the top of the list - by 'latest' we mean the ones that are created last, or the ones that have been replied last.
    • (Not Ready) The comments (the main comment, or the replies) that have not been read by the user are different in format (bolded, or colored in another color) and when the user reads them, they change the format to the basic one.
      A comment is treated as 'Read' if the user clicks on it and opens its body and the replies.
  • The section "Comments"
    • could be expanded on width if the user drags the left border to the left:
    • If all comments in accordion view could not be displayed in the section, there is a scroll on the right of the section and user could use it to view them.
  • When the user clicks on a the main comment (1) the whole thread is displayed with all the replies (2).
    • The thread is opened on left from the main comment (there could be a setting also to be opened down)
    • At the bottom of the thread there is field opened with 'Reply here' text in it (3), so the user could start replying

 

  • For each comment in the thread there is information about:
    • Comment title - the short description of the topic which is  discussed on the comment (only for the first comment)
    • User who created the comment - Detailed information about the user name, job position and department of the user who commented on the object.
    • Date and Time Stamp: The date and time when the comment has been created
    • Comment body - the full text of the comment.
  • The comments in the section are sorted by default by date of creation, and the latest are on the top.
