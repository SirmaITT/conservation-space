Sirma Platform : Search in Audit Log

The administrator is able to search in the Audit Log by different criteria using a similar to the Advanced Search form.

  • Search is case insensitive for all search criteria.
  • Most fields have auto suggest - when the user clicks in the field and types in text, the list with suggested values is being filtered on each keystroke.
  • Most fields allow selecting more than one value to be selected. The values are connected with "OR" operator.
  • For the "Event date" field a built-in calendar is implemented, allowing the user to select date and hour of the event.
  • The Context field displays the cases and projects from the current state of the semantic database. This means that if a case's title is changed, the previous title will not appear in the Context field. However, previously recorded entries for the case will remain unchanged and they may be found only by other criteria.

Example:

  • The user enters the search criteria:
    •  selects "Event Date" (1-2-3-4-5).

    • selects "Object Type" to be "Tasks" and "Projects" 

  • The user searches in the AL.
    • When the user finishes with entering data (1), the query string correspondent to the entered criteria is displayed (2).
    • The user clicks "Search" (3).
    • The record/s in the audit log corresponding to the entered criteria are displayed in the table "Found activities"(4).
    •  The results are sorted by "Event date", the newest events on the top.
    • The results are paginated and the user may define the number of results to be displayed in a page (5). Available values are: 10, 25, 50,100.
    • Each column in the table represents one criteria and the data in the table may be sorted by some of the columns ascending/ descending.  In the example sorting by this column is not active (6-7).
    • The user may select which columns to be added/ removed from the table selecting the correspondent check box from the list (6-7-8).
    • Paging is available and the user may browse the table (9).
    • There is information of the total number of records found from the search (10).

  • If no data is entered for the search criteria and the user clicks the "Search" button then the whole log would be visualized paginated in the table.
