Start Workflow from document landing page

Preconditions

  1. The user is able to start a workflow, while working on document, or directly from the list with actions on a document from the Documents dashlet on the Project or Case dashboard. 
  2. The action "Start Workflow" is context dependent and is available for specified document types and is implemented in specified sections in cases.

When the action "Start Workflow" is performed a relationship is automatically created between the document and the workflow.

  • The user is on a document landing page and the document is part of a case (1). The user selects “Start workflow” from the list with actions on the document (1-2).

    • A list with admissible workflows for the document is displayed and the user can select one.

    •   A relationship of type "Processed by/ Processes" is automatically created between the document and the workflow (1).
    •  The user fills in the necessary data and start the workflow (2-3-4).  

  • The created relations are visible in the "Relations" section.

  • The workflow is visible in "Workflows" dashlet of the case. If the workflow details page is opened for review, the relationship of type "Has child/ Part of" created between the workflow and the case is visualized.

  • When the document is part of Case A and Case B (the document is attached to case A and case B) and a workflow is started in case A, then a relationship of type "Processed by workflow/ Processes document" is automatically created between the document and the workflow, the workflow is displayed in the dashboard of the case A and a relationship of type "Has child/ Part of" is created between the workflow and case A. There is no relationship between the workflow and Case B.
  • If the document is part of a project (is not in a case) and a workflow is started, then a relationship of type "Processed by/ Processes" is automatically created between the document and the workflow, the workflow is displayed in the dashboard of the project, a relationship of type "Has child/ Part of" is created between the workflow and the project

Workflow could change the status of a document

Some of the workflows running on a document could change the document status and others could not.

  • The user is on a document landing page and the document is in status “Draft" (1). There is a relation of type "Part of" with the case where the document is added (2).
  • The user clicks "Start workflow" (3-4).

  • The user selects the workflow type (1-2). The selected type of workflow could change the status of the document when started.

  • The start task of the workflow is displayed.  The user fills in the necessary data (1-2).
  • A relationship of type “Processes document/ Is processed by workflow” is created between the workflow and the document and is visible in the relationships widget (3).
  • The user clicks "Start Workflow" (4).

  • The status of the document changes to “Pending approval” (1). (or  “In process” – this is just a label in the code list).
    •  The document is in relationship of type “Is processed by” with the current task/workflow (2).
    • The user could not edit the relationship type.

Workflow task could change the status of a document

Some of the tasks in a workflow may change the status of a document.

When the assignee of the task completes it, the status of the document in relationship with the task changes.

In the example the last task in the workflow "Publish Document" has changed the status of the document to “Approved” (1-2).
