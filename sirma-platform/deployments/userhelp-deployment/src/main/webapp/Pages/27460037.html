Sirma Platform : Suspend/ Restart Project

A project could be suspended and restarted more then once. 

  • Suspend Project
  • Restart Project

Preconditions 

  1. A project could be suspended only if in status - "Submitted" or "Approved" or "In Progress".
  2. A project could be suspended/ restarted only by an user with the role of "Project Manager".
  3. A project can be restarted only if in status "On Hold"  (if it was suspended before that). 

Suspend Project

  •   The user starts the operation from:
    • Project dashboard/ Project details dashlet/ Actions/ "Suspend Project" (1-2)
    • View Project Profile/ Actions/ "Suspend Project" 
    • Personal dashboard/ My projects dashlet/ "Suspend Project"


  • The system displays the message "Are you sure that you want to suspend the project? This will suspend all tasks assigned to the users. Yes/ No" . The user selects "Yes".

  • The system opens the project details screen where:
    • The project status is changed to "On Hold" (1).
    • If there are tasks or sub-projects/ cases on the project which are in the active state "In Process", then they are automatically suspended and their state is also changed to "On Hold".
    • The action "Suspend Project" becomes inactive and invisible and the action "Restart Project" becomes active and visible (2-3).
    • Notification is sent to the assignees for the suspended tasks.

Restart Project

  • The user selects the action "Restart Project" in the Project details screen. ( see screen above - 2-3).
    • The project status is changed back to the one it was before being suspended (1). 
    • All tasks and sub-projects/ cases that were suspended and in status "On Hold" are changed back to status "In Process". If there were tasks manually set to status "On Hold" before the project was suspended, then these tasks are automatically activated after the project restart, and the assignee may observe them.
    • The button "Restart Project" becomes inactive and the button "Suspend Project" is activated.(2)
