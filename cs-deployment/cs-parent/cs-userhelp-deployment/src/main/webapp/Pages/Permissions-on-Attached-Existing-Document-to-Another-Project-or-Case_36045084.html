Sirma Platform : Permissions on Attached Existing Document to Another Project or Case

When a document, created in one case or project, is attached to another project or case it strictly should keep its current existing permissions (either the default ones set on project/ case level, or the special ones, that are additionally set to it) and does not inherit the permissions of the project/ case to which it is attached.

Example:

  • The user opens the case details page (1) with inherited/ special permissions (2) and attaches existing document in its official section (3).

  • The attached document is in another project/ case (1). The document is with inherited/ special permissions (2-3-4).
