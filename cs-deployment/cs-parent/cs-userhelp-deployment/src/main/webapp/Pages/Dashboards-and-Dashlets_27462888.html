Sirma Platform : Dashboards and Dashlets

The section describes the requirements for the viewing and managing content in the dashboards in the system -  user personal dashboard, project dashboard and case dashboard. 

Dashboard is a central location for key portions of information associated with different primary system objects (tasks, projects, cases, users, documents, domain business objects, etc). The dashboards are context dependent : 

1. The Personal Dashboard will let  the user to view only information on objects related with him - projects on which the user participates, documents that he crated, tasks assigned to him, etc. 

 2. The Case Dashboard will let the user to view only information related to the specific case - documents and images associated with the case,  tasks which are executed on this case, workflows on the case, etc. 

 3. The Project Dashboard will let the users to view only information related with the specific project - who works on the project, what cases are in this project, activities on the project, tasks per the project, etc. 

A portion of information for one object on the dashboard is located in a pane, which will be called "dashlet". 

One and the same dashlet could be visible in all three dashboards, but with different information, based on the context.  


For example: 

"Tasks" dashlet will show only tasks assigned to a user in the user personal dashboard, and will show tasks completed/ running on a case in the case dashboard, and will also show all tasks assigned to all project members in the project dashboard.
