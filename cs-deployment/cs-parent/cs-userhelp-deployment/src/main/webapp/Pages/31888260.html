Advanced Search

Access to the Advanced search form
  • The user accesses the Advanced search from the link in the Basic Search form.

Main features of the Advanced search form
  • When the link is selected, the Advanced Search form opens with only one section added (1). Each section has only one criteria "Object type" and may represent All object types or only one object type, selectable from the drop down list (3). The list contains all object types in the system.
  • The user may add another section in the form clicking the button "Add group"(2).
  • In next row the user may select the object property to search by from the drop-down list. By default the Any field option on the top of the list is selected (4).
    • For each selected property a comparison operator is implemented. The value of the operator is selected from a drop-down list (5). The operator is different for the different types of properties:
      • For relation properties the comparison may be selected from: set to, not set to, set to some but not to
      • For string properties the comparison may be selected from: equals, does not equal, contains, does not contain, starts with, ends with, does not start with, does not end with
      • For date properties the comparison may be selected from: is, is after, is before, is between, is within
      • For numeric properties the comparison may be selected from: equals, does not equal, is greater than, is less than, is between
      • For boolean properties the comparison may be selected from: is, is not
    • The user enters value (6) for the object's property. The value is selected from the auto suggest list for the entered value of the object property of the selected object type. More then one value may be entered.
  • The user has the option to add/ remove properties from the query (7).
  • The user may expand the "Search Query" field (8) in order to review and control the entered query string which automatically appears (9) when new values are added in the search form.
    • The user may write the search query string directly in this field or paste a query string and perform search. In this case the search is performed irrelevant to the entered criteria above.
  • The search is initiated from the button "Search"(10).
  • The founded results are listed , paginated, sorted by title, descending. The user has the option to sort the results by different criteria, selectable from list, descending or ascending (11-12-13).
  • From the Advanced Search form the user may Reset  the selected values in the form, or may go to the Basic search form or Load an existing saved filter.

Examples

Example 1: Find the projects of type "Project for testing" with title containing "est", created in the period from 20 to 30 January 2015.

Select Object Type and Add Search Criteria
  • The user selects the "Object type".
    • The user opens the list, starts entering the text (1-2). The auto suggest offers the available values corresponding to the entered text as the user types.
    • The user selects the value from the list (3).

  • When the object type is selected (1) the user chooses the object property (2). The user may select properties only relevant to selected object type.
  •  The user selects the value of the comparison operator (3), starts typing the necessary text in the value field (4) and selects the value from the auto suggest (5).

  • The selected property and value (1) is entered in the query.
    • If the user clicks the button "Search" (2) the results would appear in a table view (3). The results correspond to the entered search criteria.
  • The user adds a new row  for entering search criteria clicking the button "+" (4).

  • A new row for selecting properties for the selected object type appears.
    • The relation between the two criteria is AND.
    • The user selects the criteria "Created on", selects the comparison operator "is between" and chooses the necessary date interval (1).
      • If the "Search Query" field is opened, the user may review the entered query string (2-3).
    • The user clicks "Search" (4) and the system displays the results which correspond to the search criteria (5) in the table.

  • Date properties in Advanced Search have dynamic values when the comparison operator is set to "is within" (1-2-3-4-5-6).


Add Another Group

Example 2: Find the projects of type "Project for testing" with title containing "est", created in the period from 20 to 30 January 2015 and the documents of mimetype application/msword created by the user admin admin.

  • The user expands the search with another object type adding another group by clicking the button "Add group" (1).
  • New empty group is added in the form where:
    • The relation between the groups is OR
    • The user selects the object type (here - "Document").
      • Two or more groups may have same value for "Object Type"
    • The user defines the properties for the selected type and its values (3).
      • The group may be removed from the search by clicking the button "X" (4).
  • The user clicks "Search" (5).
  • The founded results (6) correspond to the selected object types, properties and values, entered in the form.

Combine Search Criteria with OR

Example 3: Find the projects of type "Project for testing", created on 13 January 2015 and all projects of type "Project for testing" with title beginning with "low" (no matter when they were created).

The user has the option to combine the search criteria in the advanced search form with OR relation. Such relations refer only within the group.

  • The user selects the object type from the drop-down menu to be "Project for testing"
  • The user selects the search the property "Created on", chooses the comparison operator to be "is", and selects the value from the calendar to be the date 01/13/2015 (1).
  • The user adds another row, selects from the drop-down menu the relation between the search properties to be "OR" and fills in the necessary data (2).
  • The user initiates the search (3).
  • The founded   results (4) correspond to the selected object type, properties and entered values.

Combine Search Criteria with Brackets, AND and OR
  • The user has the option to use brackets () in the advanced search form and group his/ her search criteria.
    • The brackets refer only within the group.
    • The drop down list for brackets is with up to 3 brackets (opening/ closing).
    • The user is notified if there are missing brackets.
  • The user has the option to combine search criteria in the advanced search form with AND relation or with OR relation.

Example 4:

In the system there are projects of different types ( "Project for testing", "Project1", etc) in different status ( "Submitted", Approved", etc).

a : The user searches  for projects of type "Projects for testing" in status "Submitted", created after 02/01/2015 and all types of projects in status "Approved"

  1. The user enters the search criteria.
  2. The user applies the necessary operators (AND / OR).
  3. The user initiates the search.
  4. The results are displayed. They correspond to the entered criteria in statements.

In the results are displayed 7 projects:

  • 5 projects of type "Project for testing" in status "Submitted"
  • 1 project of type "Project for testing" in status "Approved" (1)
  • 1 project of type "Project 1" in status "Approved" (2)

Note: Both projects in "Approved" are displayed in the results list irrelevant to to entered criteria "Created on" due to the order of entered criteria in the search statement.

b : The user searches  for projects of type "Projects for testing" in status "Submitted", created after 02/01/2015 and all projects of the same type "Projects for testing" in status "Approved"

  1. The user enters the search criteria.
  2. The user applies the necessary operators (AND / OR).
  3. The user inserts brackets (1-2) on the necessary places, selecting them from the corresponding drop-down menu.
  4. The user initiates the search.
  5. The results are displayed. They correspond to the entered criteria in statements.

In the results are displayed 6 projects (3):

  • 5 projects of type "Project for testing" in status "Submitted"
  • 1 project of type "Project for testing" in status "Approved" (4)

Note: The project in "Approved" is displayed in the results list irrelevant to to entered criteria "Created on" due to the order of entered criteria in the search statement.
