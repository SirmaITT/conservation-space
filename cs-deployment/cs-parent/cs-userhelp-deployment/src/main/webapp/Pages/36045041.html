Sirma Platform : Manage Special Permissions on Enterprise Objects (context dependent)

User with role Administrator, Manager, Collaborator, Contributor on an enterprise object (case, document, image, iDoc) is able to manage the special permissions on the object to some users and groups in addition to inherited permissions from its parent object.
The special permissions are always with higher priority then the inherited permissions.

For example an user is with the role Consumer in the Inherited permissions  section will have more permissions if he is given the role of Collaborator in the special permissions.

Example:

  • User with relevant permissions opens a document created in a section of a case with Inherited permissions (1-2-3-4).
  • The user clicks the button "Edit permissions" (5).

  • The section Special permissions opens in edit mode.
    • The user adds a new user or group  and selects role for it list (1-2).
    • The user add special permissions to user that has already inherited permissions (3-4). The inherited permissions are of a Contributor, the selected special permissions for the same user are of a Contributor.
    • The user may change user/ group, may select new role for them or remove a whole entry clicking the corresponding button "-" (5).
  • When ready the user saves the added/ edited permissions (6).

  • The changes are saved:

    • The Permission type of the object changes to Inherited and Special (1).

    • The added/ edited permissions appear in the section Special Permissions (2). The user Maria Todorova has the role of Collaborator (3) for this object, not of a Contributor as shown in section Inherited Permissions  (4) because the permissions in the Special Permissions section are with higher priority then those in section Inherited Permissions.
    • The permissions in section Inherited Permissions are not changed.
    • The permissions in section Management Permissions are not changed.
