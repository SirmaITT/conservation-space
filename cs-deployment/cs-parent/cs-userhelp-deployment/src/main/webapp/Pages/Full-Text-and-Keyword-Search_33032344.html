Sirma Platform : Full Text and Keyword Search

The Full Text and Keyword Search is used to search and access objects in the system no matter of the context the user is working on. The list of founded results displays only the objects that have in their content (title, description, content, or other indexed properties) the text that is typed. File format for content searching is: .txt, .doc, .xls, .pdf, .ppt, .html, .rtf, .xml

  • The user types in text in the search box (1).
    • A list with auto suggest values is displayed under the search box (2).
      • The list displays only the objects that have in their content (title, description, content, or other indexed properties) the text that is typed.
      • The list displays no more than 20 (parameter) of the most relevant search result values.
      • The list changes automatically with every next symbol that is typed in the search box.
      • If the user selects one of the automatically suggested objects, then he/she is transferred to the selected object landing page.
    • If no text is entered in the search box (1) and the user selects the search action next to the search box (3), the basic search screen will be displayed AND all search criteria will be empty.
  • After the text is entered, the user clicks the key "Enter" from his/ her PC keyboard.

  • The system opens the basic search screen where:
    • The text in the search box is copied in the text field in the basic search (1).
    • The search result is displayed (2)

  • If the user passes over the search field with the cursor a tool-tip appears with main rules to perform searches from this field.

  • The user is able to filter the results from the search (1) using more search criteria from the basic search form (2).
  • The full text and keyword search option cannot be used if the Advanced Search option is selected (3).
