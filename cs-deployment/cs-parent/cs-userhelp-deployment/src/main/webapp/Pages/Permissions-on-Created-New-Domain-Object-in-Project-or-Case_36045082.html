Sirma Platform : Permissions on Created New Domain Object in Project or Case

When a new  domain object  is created in a project or in a case, the created object inherit the permissions set on library level, not the permissions from the context it is created in.

If the user who wants to create a new domain object in a project/ case does not have relevant permissions for that object library, he is not able to see this library in the list with objects and cannot create such an object.

Example:

  • In the Domain Object library "Mark" the permissions are set: admin admin - MANAGER, Dimitar Glezev - Contributor, SYSTEM_ALL_OTHER_USERS - CONSUMER

  • The user Daniela Todorova creates a project with title Test Project 21March. Her role in the project is MANAGER.

  • The Manager of the project with title Test Project 21March, tries to create a new object of type "Mark" in the project selecting the action from the project's details page.
    • In the drop-down list of Objects library  "Mark" library is visible (1-2) (the consumer permissions set on this library are for all users).
  • In the drop-down list with available for creation object types the type "Mark" is missing because this user (Daniela Todorova) does not have permissions to create objects in this library.

  • If the manager ob Domain Object library "Mark" gives relevant permissions to this user (Daniela Todorova) for the library:

  • Then this user (Daniela Todorova) may create objects of this type ("Mark") in His/ her projects/ cases.
