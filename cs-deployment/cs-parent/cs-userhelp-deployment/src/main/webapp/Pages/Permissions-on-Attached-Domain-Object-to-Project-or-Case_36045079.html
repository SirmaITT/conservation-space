Sirma Platform : Permissions on Attached Domain Object to Project or Case

When a domain object is attached to a project or case, it strictly keeps its existing permissions (either the inherited from the library, or the special ones, that are additionally set to it) and does not inherit the permissions set to the project or case it is attached to.

Example:

  • The user opens a case with title "Test Case 1" with inherited permissions (1-2-3).

  • In the domain library "Mark" there is an object with title "Test 18March-Mark", with inherited library permissions (1-2-3). User3 is a contributor for this domain library.

  • The user attaches the object with title "Test 18March-Mark" in the objects section of the case with title "Test Case 1" (1).
  • The user opens the attached object landing page. The permissions on the attached object remain unchanged and are the same as the inherited library permissions (2-3-4) of the object before the attachment.
