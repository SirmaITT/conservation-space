Sirma Platform : Stop Case

When the process "Stop Case" is completed for a case in an active state, the case becomes not active. No further action can be performed with the case except for:

  1. relations from other open cases to the closed case
  2. relations from attached documents in other open cases to documents in the closed case

Preconditions

The action "Stop Case" may be performed only for cases in an active state. 

  • The user starts the operation from:
    • Personal Dashboard/ My Cases dashlet/ Actions/ Stop Case for a chosen case
    • Case Dashboard/  Actions/ Stop Case
    • Case Dashboard/ Case details dashlet/ Actions/ Stop Case (1-2)
    • page Case details/ Actions/ Stop Case

  • The system displays a confirmation massage. The user clicks the button "Yes".

  • The system opens the case dashboard. 
    • The case details "Completed on:" and "Completed by:" are updated automatically (1).
    • The case state changes to "Stopped"(2).
    • All active tasks for the case are stopped and disappear from the Active Tasks filter and may be seen in filter "All Tasks" (3-4).
    • All workflows for the case are stopped and may be seen in filter "All Workflows"  (5-6).
    • The added documents in the case may only be previewed and downloaded and no changes with them are permitted (7).
    • All objects added in the case may only be cloned (8).
    • Cases in state "Stopped" may only be previewed and no other actions with them are permitted (9).
