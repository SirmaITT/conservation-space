Sirma Platform : Manage Special Permissions on Domain Objects (context independent)

Users with role Administrator, Manager, Collaborator, Contributor on a domain object are able to manage the special permissions on an object for some users and groups, in addition to the inherited library permissions on the object.

Example:

  • The Manager of domain library "Mark" (1) has added permissions (2) to:
    • AnnCollaboratorsGroup - Collaborator (3)
    • Daniela Todorova - Contributor (4)

  • The user AnnCollaborator (1), member of the groupÂ  AnnCollaboratorsGroup, opens the landing page of an object from "Mark" object library (2).

    • The user opens the permissions tab (3) of the object.
    • The user is able to add Special permissions on the object which is with Inherited permissions. He clicks the button "Edit permissions" (4) .

  • The section Special permissions opens in edit mode.
    • The user adds a new user or group (1) selecting it from list (2).
    • The user selects role of the new user/ group from list (3).
    • The user is able to remove the entry (4) or add another one (1).
    • The user may change existing user/group (2).

    • The user is able to change the existing/ selected role of user/ group (3)

  • When ready the user saves the added/ edited permissions (5).

  • The changes are saved:

    • The Permission type of the object changes to Inherited and Special (1).

    • The added/ changed permissions appear in the section Special Permissions (2).
    • The permissions in section Inherited Permissions (3) are not changed.
    • The permissions in section Management Permissions (4) are not changed.
