Sirma Platform : Recall Workflow

The creator of a workflow (the user who starts the workflow), is able to recall the workflow back to themselves at any point of time, provided that the workflow is In Progress.

To "recall" a workflow means to start it from the beginning. The action may be performed in case the creator changes their mind and wants to reassign the workflow tasks to different user/s and still to keep the completed tasks in the workflow.

Example:

  1. The user Regular User starts a workflow on a document (My iDoc 1) that was created in a project's case and assigns the first workflow task (Review and approve document by second reviewer ) to the user (at_project_manager).
  2. The user at_project_manager approves the first workflow task (Review and approve document by first reviewer ). The assignee of the next task is again at_project_manager.



  3. The user at_project_manager approves the second workflow task and is the assignee of the third workflow task (Publish document).



  4. The user Regular User, as creator of the workflow, decides to reassign the workflow tasks to a different user. That is why Regular User selects Workflow Recall action for the workflow.

  5. A message about the workflow recall is displayed.



  6. The user Regular User selects a different user (at_collaborator)  as assignee for the first workflow task and restarts the workflow.



  7. The completed and/or stopped tasks are kept in the Completed Tasks table of the workflow and the first task is assigned to the new user.



  8. The action is logged in the Audit Log
