Sirma Platform : Restore All Entities to Library Permissions

The Administrator in the system is able easily to restore the permissions of all entities in a library to the current library permission and thus to remove all existing special permission on single objects, in case there is a need to change to security policy in the organization.

Rules:

  1. There is "Restore children permissions" action for every object in the library that has children, including the libraryÂ  itself.
  2. The action is visible only when Edit Permissions is selected for the library or respective object.
  3. Only the user with management role can restore permissions.
  4. When the action "Restore children permissions" is selected, then a message is displayed "All special permissions of the children objects will be removed! The action is non-reversible! Are you sure that you want to continue? Y/ N"
  5. If the user selects Yes then all permissions of the children objects are set to Inherited, and the special permissions are removed.
  6. The action Edit permissions for all objects with restored permissions is logged in the audit log (Edit permissions is performed only for objects with "Special" or "Inherited and Special" permissions. For objects with only Inherited permissions no changes are made.)
  7. The permissions of the library or of the parent object, from which is selected "Restore children permissions" are not changed.

Example:

  • In a project (1) Special permissions are set for a created object (2-3-4-5).

  • The Administrator/ Manager of the object's library decides to restore the permissions of all objects in that library to the default permissions and remove all special permissions set on objects from this library.
    • The administrator/ manager opens the library details and selects the action "Edit Permissions" (1-2-3).

    • The administrator/ manager selects the action "Restore children permissions"

  • The system opens a confirmation message and the administrator/ manager confirms the operation (1-2).

  • The special permissions set on the object are removed (1-2-3) and the permissions on all object in the library are set to the default permissions.

  • The actions are logged in the Audit Log.
    • when the button "Restore children permissions " was selected (1).
    • when the special permissions of the object are removed (2).
    • when the special permissions of other objects from the same library are removed (3).
