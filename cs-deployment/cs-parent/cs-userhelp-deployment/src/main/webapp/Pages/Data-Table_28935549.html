Sirma Platform : Data Table

  • Select Data Table widget
  • Select "Object to be displayed"
    • Option  "Search for object"
    • Option "Manually select objects"
    • Set up the widget display option
  • Select "Object details to be displayed"
  • Manage Data Table
    • Add and Remove row only for manually selected objects
    • Set up additional (free text) columns that will be added into the data table (NOT READY)
    • Sort content, Display or Hide column
    • Edit, Add, Delete object details

The user is able to insert and manage a Data Table widget into an iDoc.

Constraints

  1. Objects from different types could be displayed in the table.
  2. One and the same object could exist only once in a data table. (Check by object ID)

Select Data Table widget

  • The user inserts the Data table widget from the widget palette into an iDoc  which is in edit mode (1).
  • The data table widget is inserted empty into the iDoc (2). The user adds a title (3) and selects the "Manage Properties" button in order to set up the widget (4-5).

Select "Object to be displayed"

  • The system opens a screen where the user could select one of the following sections:
    • Objects to be displayed (1) - selected by default. In this section the user selects the objects that will be displayed in the widget  (the user could select one or more objects).
      • Manually select object (3)
      • Search for object (4) - selected by default
    • Object details to be displayed (2) - in this section the user selects the details of the objects he/she wants to be displayed.

Option  "Search for object"

  • The option "Search for object" (1) is selected. The user defines the search criteria: 

    • Using the option to select objects from the current context:

      • Object type (here Document(2)) -  the user could select one or more object types he/she is expecting to be displayed in the widget.

      •  Search context (here Current case (3)) - from which the objects to be extracted
      • The user clicks Search (4).
    • Using any of the search criteria (Create From, Create To, Create By, Contains text ...)

If no object type is selected then all the objects from the context will be displayed in the widget.   If the user selects value 'Any type' this means that in this widget to be displayed information for any first class object regardless of its type. In this case the system will display only the common objects attributes, like Type, Title, Description, Created by, etc.

If the user selects only one first class system object  to be displayed (project, case, task, document, specific domain object, etc.) in the widget then the system displays the sub-types of the selected object that exists plus value Any type.

    • The system displays the results (5).
    • The user selects "OK" (6).

  • In the data table are displayed all objects that satisfy the criteria. The user does not have to care about new objects that satisfy the search criteria, or objects that do not satisfy the criteria any more.

The widget is updated every time the iDoc is reloaded and if a new document is added/ deleted in that case - it will be displayed in the widget.

 

Option "Manually select objects"

If the user selects this option he would be able to browse and search for any object in the system and select it for display in the widget.

  • If the user selects "Manually select objects" (1) - then he/she  could select manually only few of the objects in the search result, and they will always be displayed in the widget no matter if they continue to satisfy the initial search criteria or not.
    • The user selects the button "Search" (2) for selecting and adding an object from the whole system for which he/ she has permissions.
    • The user may select the button "Browse" for selecting and adding an object from the tree view of a selected project for which he/ she has permissions. By default the current project is selected.
    • The user may select the button "Upload" for selecting and adding an object from his/ her PC.

  • The selected objects are inserted in the table (1). The user selects from the drop down menu (2) the "Manage Properties" action.

Set up the widget display option

The user could set up the behavior of the widget in document Preview mode:

There is a check box  in the "Data table" screen with label "Hide if no objects are found".

If the check box is selected, then in the document preview mode the widget will be hidden if no object is found,  to be displayed in the widget.

 

Select "Object details to be displayed"

  1. If objects from one object type are selected, then in this section in the widget are displayed all metadata for the selected object type.
  2. If objects from different object types are selected (as in the example above), then in this section in the widget are displayed only the common metadata for all objects (title, description, type, date created, date updated, etc.)
  • The user selects "Object details to be displayed" (1) and selects the check box of each attribute of each type of object to be displayed in the widget:
    • selects the data for the Case (2-3-4)

    • selects the data for the Document (1-2)

  • selects the data for the Task (1-2) and clicks the "OK" (3) button.

  • The system displays the document in edit mode and the user saves the document.
    • The selected objects appear in the table (1). The object ID is represented as a link to the object. The user could click on the link and the object from data table will be opened.
    • The table has columns titled by each selected property (2)
    • There is a drop down menu for each column and the user is able to sort the data in the column ascending or descending (3). 
    •  If the user selects "Columns" (4), the columns in the table are displayed (5).

 

Manage Data Table

Add and Remove row only for manually selected objects

If already some objects are displayed in a data table, then:

  • The user could perform additional search and select more objects to be displayed (1) and select object from another page (2).
  • The user could delete some of the table rows (3) by selecting the correspondent "Remove (4) button and that object will not be displayed in the widget.
When a row is deleted from a table the object itself is not deleted.
  • The user saves the changes (5).

 

Set up additional (free text) columns that will be added into the data table (NOT READY)

The user will be able to add columns in the table, which are not part of the selected object properties. The column will be initially empty and the user will be able to insert data into it.

This additional column could be added only into data tables with manually selected objects for display.

Example: for the purposes of created report the user has to check for each object in the table if an analysis is completed or not, and there is no such properties in the object metadata. The user will create a new column in the table with the name "Completed analysis (Yes/ No)" and captures manually for each object instance the result of the analysis. Sort content, Display or Hide column

In each column there is a drop down menu (1). The user may:

  • sort the objects in the table ascending or descending by this column
  • hide or display columns (2-3)
    • the user deselects a column from the drop down menu where all columns in the table are visualized. If the column is with object properties then removing it from the table will only hide its content.

This is useful when there are many columns in the table (many or all details were selected to be displayed) but for the moment the user wants only a few to be displayed.

 

      • The column "Created by" is hidden.
When the check box in front of a property is checked - this column is visible in the table.
  • Order the columns in the table by drag and drop (1-2-3).

The user selects a column with the right button of the mouse (1), holds it, drags the column (2) to the new location and drops it (3).

Edit, Add, Delete object details

If a column in the data table is associated with the object properties allowed for editing - the user is able to add/ edit/ delete object details displayed into this column.


The behavior of a field in the data table is exactly the same as defined in the object definition.
  • If a property is an editable data field, then it will be displayed in the table as editable data field.
  • If a property (1) is value selected from a drop down list (2), then it will be displayed in the data table as a drop down list, and the user is able to change its value.
