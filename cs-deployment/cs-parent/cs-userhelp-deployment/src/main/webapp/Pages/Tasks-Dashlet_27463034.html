Sirma Platform : Tasks Dashlet

The Tasks dashlet is context dependent and the content displayed on it depends on whether the dashlet is in the Personal, Project or Case dashboard. 

  1. On the Personal dashboard the Task dashlet displays only the tasks that are assigned to the specific user.
  2. On the Project Dahsboard the Task dashlet displays all the tasks assigned to the project resources of this project.
  3. On the Case dashboard the Tasks dashlet displays the tasks which are part of the case.
  • In the header of the dashlet is displayed the number of projects in the dashlet and the total number of all projects of the same type to which the user has access (1). 
  • The tasks in the dashlet are sorted by default by date of creation and the latest ones are on the top. For each task in the Task dashlet is diplayed: Task ID, Task Name, Task Type (optional), Task Status. They all are a link (2) to the corresponding task landing page.
  • The tool tip (3) of each task displays more information about the task (Assigned to, Created on).

  • The items in the dashlet may be sorted by the following criteria, selectable from the drop down list (1-2):
    • Date modified on - this is the default sort criteria, descending order (the items recently created/ updated are on the top)

    • Title

    • Priority

    • Planned Start date

    • Planned End date

    • Status

  • For each combination of criteria the user is able to sort the items in the dashlet in ascending or descending order (3).

On Personal Dashboard/ My Tasks Dashlet are displayed only tasks in “active” status  e.g., Submitted, Approved, In Progress, Suspended.

  • The following filters (1-2) are applied on the dashlet:
    • Active tasks (default) - this is the default filter and when selected the user sees all tasks in Active state (Submitted, In progress, Approved, Suspended) which are assigned to him/ her or are created by the current user, including unassigned (not claimed yet) pooled tasks and the current user is in the group of users of the task or creator of the task.
    • Tasks assigned to me - the filter displayes only tasks that are in active state and are personally assigned to the current user.
    • Unclaimed tasks - this filter displays only the pooled tasks, in which the current user is a member of the group assigned to the task but no claim has been made yet, or the current user is creator of the task and the task is unclaimed.
    • High priority tasks - the filter is the same as Active tasks (tasks assigned to the current user, created by him/ her or unassigned pooled tasks) but displays only the tasks with priority "High"
    • Due Today tasks - this filter is the same as Active tasks (tasks created by the current user, assigned to him/ her or unassigned pooled tasks) but the task due date is today.
    • Overdue Tasks - this filter is the same as Active tasks (tasks created by the current user, assigned to  him/ her or unassigned pooled tasks) but the task due date is prior today.
    • My Favorites - shows all the items that the user has marked as favorites (only on My dashboard).

On the Project Dashboard/ Tasks Dashlet are displayed only tasks created in the project or in cases in the project.

  • The following filters (1-2) are applied on the dashlet:
    • Tasks assigned to me (default) - this is the default filter and the filter displays only tasks that are in active state and are personally assigned to me
    • All active tasks in the project -  when selected I see all tasks in Active state (Submitted, In progress, Approved, Suspended) which are assigned to me or to other project members, including unassigned (not claimed yet) pooled tasks.
    • Unclaimed tasks - this filter displays only the pooled tasks in the project, which have not been claimed yet
    • High priority tasks - the filter is the same as Active tasks, but displays only the tasks with priority "High"
    • Due Today tasks - this filter is the same as Active tasks, but the task due date is today
    • Overdue Tasks - this filter is the same as Active tasks, but the task due date is prior today
    • All tasks - the filter displays all task in the project (both in active state and in completed and stopped state).

On the Case Dashboard/ Tasks Dashlet are displayed only tasks created in the in case.

  • The following filters (1-2) are applied on the dashlet:
    • Assigned to me tasks (default) - this is the default filter and the filter displays only tasks that are in active state and are personally assigned to the current user.
    • All active tasks in the case - displays all tasks in Active state (Submitted, In progress, Approved, Suspended) which are assigned to the current user or to other project members, including unassigned (not claimed yet) pooled tasks.
    • Unclaimed tasks - this filter displays only the pooled tasks in the case, which have not been claimed yet.
    • High priority tasks - the filter is the same as Active tasks, but displays only the tasks with priority "High"
    • Due today tasks - this filter is the same as Active tasks, but the task due date is today.
    • Overdue Tasks - this filter is the same as Active tasks, but the task due date is prior today.
    • All tasks - the filter displays all task in the case (both in active state and in completed and stopped state).


  • Next to each task in the dashlet there is a list with available Actions on the task (1), which are restricted by user's role in the Project and Case. In general the actions include: Edit Task Details, Create Sub-task, Reassign Task, Suspend Task (2).
  • No actions are available for completed tasks.


  • From the Project Tasks dashlet in the Project dashboard and  in the Case dashboard/ Tasks dashlet the user could start a new task selecting the action "Create Task" from the list (1-2).

The action is available for users with corresponding permissions.
