Sirma Platform : Every New Object Inherits the Permissions of its Parent

When users with the role "Manager" or "Collaborator" or "Contributor" in a project create in the project object of type content (could be case, document, iDoc, image), the created objects inherit the permissions set to its parent.
Tasks and workflows do not inherit permissions - for them exclusive permissions are given to the task assignees and task creators.

  • Create new case in project with Permission type "Inherited"
  • Create new document in case with Permission type "Inherited"
  • Upload new document in case with Permission type "Inherited and Special"
  • Upload document in case with Permission type "Special"
  • Tasks and workflows do not inherit permissions. Exclusive permissions are given to the task creators and the task assignees
  • Cloned or Moved existing document in other case with flag "Inherited and Special permissions" inherits the permissions of the parent case.

Create new case in project with Permission type "Inherited"

  • In a project with selected project members (1-2) the user creates a new case (3-4).

  • When the case (1) is created it inherits the permissions set on the project. The Permission type is "Inherited" (2-3-4-5).

Create new document in case with Permission type "Inherited"

  • The user opens the Documents and Media tab of the case and creates a new document in the official section.

  • When the document (1) is created in the case it inherits the Inherited  and Management Permissions set on the case (2-4-5).

    • The Permission type is "Inherited" (3).

Upload new document in case with Permission type "Inherited and Special"

  • The user opens the case details page of a case and gives special permissions to a selected user on the case (1-2-3).
    • The check box "Enable inherited permissions" is checked (4).
  • The user clicks Save Permissions (5).

  • The permissions on the case are saved (1).
    • The Permission type is changed to "Inherited and Special" (2).
    • The special permissions given to user appear in the corresponding section (3).

  • When the user uploads/ creates a document in the case with inherited and special permissions:
  • The uploaded/ created document is with Permission type Inherited (1-2-3). 
    • The document inherits the Inherited and Management permissions set on the case.
    • The Special permissions set on the case are not inherited on the document (4).

Upload document in case with Permission type "Special"

  • The user edits the permissions of a case (1-2).

  • The user gives special permissions to selected user (1-2-3), disables the inherited permissions (4) and saves (5).

Note: If the user wants to disable the inherited permissions he should first add at least one user/ group in the section Special Permissions and then disable the Inherited permissions.

  • The case is set with Permission type - "Special" (1) and in the corresponding sections the selected users with corresponding roles are displayed (2-3-4).

  • The user uploads a document in the official section of the case with Permission type - "Special".
  • The user opens the uploaded document (1). In the Permissions (2) section are displayed:
    • The Permission type of the uploaded document is "Inherited".
    • The section Special Permissions is empty (4).
    • The special permissions set on the case (here - given to user Alice Beecher) are transferred in the document as Inherited Permissions (5)
    • The Management Permissions on the case are inherited in the document (6).

 

 

Tasks and workflows do not inherit permissions. Exclusive permissions are given to the task creators and the task assignees

  • The user starts a workflow in case.
  • The user opens the workflow details page or the task details page (1).
    • No inherited permissions are set on the workflow or on the task.
    • Exclusive permissions are given to the task assignee (2) and to the creator of the task.

Cloned or Moved existing document in other case with flag "Inherited and Special permissions" inherits the permissions of the parent case.
