Sirma Platform : Confirm Reading

The action "Confirm Reading" on a document is used by the users, who are obliged to read a document and explicitly confirm that they have read it. The action is logged into the audit log, so the administrator can make reports about the users who have read specified documents.

Rules:

  1. The document types for which the action "Confirm reading" to be visible are configurable.
  2. The user can set up the status of the document in which the action to be active.
  3. The user can set up permissions on the action, for example users with Consumer roles not to be able to execute it and only the Collaborators and Contributors to be able to perform it.
  4. The user can set up if the action to be performed on draft document or on the revision of the document.
  5. Every user can select the action as many times as he wants (if he has permissions).
  6. After the action is selected, there is a notification message displayed to the user "The operation is logged into the system! (OK)"
  • The user selects the action "Confirm Reading" from the document's list of available actions (1-2).

  • The system displays an information message. The user clicks "OK".

  • The action is logged into the audit log.
