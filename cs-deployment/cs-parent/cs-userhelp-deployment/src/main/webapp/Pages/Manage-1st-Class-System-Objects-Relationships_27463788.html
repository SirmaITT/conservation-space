Introduction

Here are described the requirements for creating and managing references from one primary object to other primary system objects like documents, sub-projects/ cases and domain specific objects.


Only the creation and management of user defined relations are discussed, which are different than the ones created automatically by the system as "is Part of" (for example the sub-projects/case that are created into a project, or documents uploaded/ created into a project are automatically linked to this project).


  Two types of labels are supported for each reference:

  • primary reference name - this is the label that will be visible in the primary object screen (the screen of the object from which the link is created)

For example: Project A "is initiated based on" Document B.

  • secondary reference name - this is the label that will be visible in the secondary object screen (the screen of the object to which the link is created)

For example: Document B "initiates" Project A.

 

There are two types of relations in the semantic ontology of the project - simple and complex. The Relationship widget shows both the simple and complex types of relations.

When there is an inverse relation, for example there is a relation 'x' from object A to object B and relation 'y' from B to A, then in the Relationship widget of object A only the relation 'x'  is seen because object A is a source(domain).

 

Relationships tab

Except for the task and the workflow, every primary object has a Relationship tab. It is located in the object's landing page or details page, along with and next to the other tabs: Comments, Properties, and Versions. It shows all the relationships that the primary object has with the other objects.

 

For example in a case (My case 1) a document (My Document 1) and an object (My Object 1) can be created; also a workflow (Unlimited Task Assignment (UTA) workflow) and a standalone task (My Standalone Task 1) can be started. A relationship between My case 1 and another case is manually created.

The Relationships tab can be accessed in the case landing page.
