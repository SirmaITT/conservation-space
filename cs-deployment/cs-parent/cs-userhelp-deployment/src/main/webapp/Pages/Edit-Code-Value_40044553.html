Sirma Platform : Edit Code Value

Only user with administrator permissions has active option to edit a code value.

  • The user opens a valid codelist (1) and selects the code value (2) which should be edited.
  • The system opens the code value details screen for the selected code value (3). The "Edit code value details" (4) option is active only on the preview page of the code value details of a currently valid code value or a code value with validity period in the future.

The user selects the button "Edit code value details" (4)

  • The code value details are displayed in editable form. The user may enter/ edit data in the white fields (1-2).

Applied validation rules on the code value edit form:

  1.  the code value code cannot be changed (the field is not editable on the form)
  2.  the validity start date is set by default to the current date and time is round up to next whole hour (eg. 09:00, 10:00, ..., 19:00, 20:00, etc.)
  3.  the validity start date field is editable on the form
  4.  the validity end date is not changed by default - if it was null, it remains null; if it was set, its value is kept and not chaged by the system
  5.  the validity end date field is editable on the form
  6.  the validity start date must be later than the current date and time plus the next whole hour
  7.  the validity end date must be later than the current date and time plus the next whole hour
  8.  the validity end date must be later than the validity start date
  9.  the masterCV field contains only one value; the field is displayed as a drop-down with all valid code values from the master CL of the code list
  10.  the masterCV, if entered, must be a value from the masterCL of the codelist to which the current code value belongs
  11.  the validity period must not overlap with a validity period of any other record for the code value
  12. the description in English is required field

If there are validation errors on the form, the Save button is not active and data cannot be saved.

  • The administrator selects to save the changes for the code value (3)
    • The administrator may select the cancel option to revert the changes made. The system displays a confirmation dialog form.
      • If the administrator confirms the operation the system reverts all changes and the code value details form is displayed in preview mode.
      • If the administrator does not confirm the operation system returns to the code value edit form with latest changes made.

  • The system displays a confirmation dialog form (1) with two options:
    • If the administrator confirms the operation (2), the system saves the updated code value and the code value details form is displayed in preview mode.
    • If the administrator does not confirm the operation (3), the system returns to the code value edit form with latest changes made.

  • The edited code value is saved in the codelist.
    • the old code value which is valid in the current moment is visible in the list with code values (1). Its validity is changed: the Valid to date is set to the moment when the edited code value will become active (2).

    • when the edited code value becomes valid, it appears in the list of valid code values for the codelist (1-2-3).
