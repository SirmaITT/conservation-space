Sirma Platform : Change the Permissions of the Creator

The creator of an object looses his/ her absolute (implicit) permissions on the object, when he/ she is given special permissions of a consumer or is with no permissions.

Example:

The creator of a document cannot edit the document if he is with special permissions of a consumer in project/ case where the document is.

  • The manager of a project named PR16Apr gives user3 permissions of a collaborator in the project.
  • The manager of the project creates a case named Test Case 1 in the project.
  • The collaborator (user3) creates an iDoc named Doc Created by user3 in the official section of the case (1-2-3-4).
    • In the Object data widget is displayed the creator of the document (5).
    • For the created document user3 is with inherited permissions (from the case) of a collaborator (6).
    • User3 is able to edit the document and is able to perform a wide range of actions on the document (7-8).

  • The manager of the project sets for user3 Special permissions of consumer on that case (1-2-3).

  • When user3 (Dimitar Glezev) opens the document which was created by him (1-2-3).
    • User3 inherits the special permissions set in the case and now is with consumer permissions (4)
    • User3 is NOT able to edit the document and is able to perform a limitted range of actions on the document (5-6).
