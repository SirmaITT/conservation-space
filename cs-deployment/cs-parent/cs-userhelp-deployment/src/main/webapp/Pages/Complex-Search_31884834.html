Sirma Platform : Complex Search

Member of the administrators group may perform complex searches in the audit log.

The user is logged in and the Audit Log page is opened.

  • Complex search with AND
    • Exclude some entries from the results (NOT)
  • Complex search with OR, AND+OR, brackets

Complex search with AND

Example: In the system there are object/ s with object title containing "test" and object type "Task" and there are actions performed in the period 01-10 October 2014.

  • The user specifies the criteria with AND:
    • selects the period (1-2)
    • selects the value of the operator to be "AND" (3) for search criteria "Object type" , does not change the comparison operator "in" and selects "Task" (4) in the search field
    • selects the value of the operator to be "AND" (3) for search criteria "Object title" , sets the comparison operator to "contains" (5) and enters "test" (6) in the search text field
  • The query automatically appears in the query field (7)
  • The user clicks "Search" (8)
  • The system displays the results in the table (9) - the results correspond to the selected criteria values.

Exclude some entries from the results (NOT)

Note: The same data as in the example above is used.

  • The user specifies the criteria with AND:
    • selects the value of the operator to be "AND" for search criteria "Action", sets the comparison operator to "not in" (1) and selects the action "Edit Details" (2) in the search field.
    • selects the period
    • selects the value of the operator to be "AND" for search criteria "Object type", sets the comparison operator to "in" and selects "Task" in the search field.
    • selects the value of the operator to be "AND" for search criteria "Object title" , sets the comparison operator to "contains" and enters "test"  in the search text field.
    • the query automatically appears in the query field (3)  
  • The user clicks "Search" (4) 
  • The system displays the results in the table (5).

The list of search results displays all recorded actions except recordss for actions "Edit details" which are excluded from the search results list.

Complex search with OR, AND+OR, brackets

Example: The report must display all actions "Create new project" for user "admin.admin" in the time period from October 10 2014 till the current date/ OR all "Tasks" with title containing "mir" in the time period from October 01 2014 till the current date.

  • The user inserts/ selects the search criteria.
    • Helpful Tips:
      • Carefully select the order of the criteria in the query.
      • Use brackets only if necessary (1) - (5) , selecting them from the corresponding drop down menu.
      • Select the operator OR (6) when need to group search criteria. Depending on the query the system automatically inserts brackets in the query string if needed, when using the operator OR.
      • Add rows (10) if needed and select criteria (11).
  • The user clicks "Search" (12)
  • The results are displayed in the table (13) and correspond to the set criteria.

  • If there is an error in the used brackets - the wrongly inserted bracket is in red (1) and no search query is generated (3).
    • When the correct (here - closing) bracket is inserted (2) the circled in red bracket field will turn in black and the query will be generated.
