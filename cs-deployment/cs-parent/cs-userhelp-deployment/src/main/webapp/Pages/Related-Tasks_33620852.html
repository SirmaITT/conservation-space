Sirma Platform : Related Tasks

The "Related Tasks" widget by default displays only objects of type "Task". The user could create directly from the widget new tasks in a selected context.
  • Manage tasks in the current context.
    • Changing the default settings for new tasks
  • Create new task from the widget in the current (selected) context.

Rules:

  1. When the widget is inserted by default is executed the following query: Show me all tasks related with the "current project".
  2. In the widget by default are selected the following task properties to be displayed: Assigned to and Due Date (plus the compact header of the task represented as a link in the first column)
  3. The user is able additionally to set up the widget by changing the query - to search for specific task types to be displayed, to change the context, etc. (only tasks and different types of tasks could be displayed)
  4. The user is able additionally to set up the task properties to be displayed in the widget - to remove the default and to add new, etc
  5. The user is able to set up some default values when a new task is created - the task type, the task context and the task assignee - this will simplify the creation of the task.

Manage tasks in the current context.

  • The user opens for edit an iDoc in a project and insert the Related Tasks widget in its content section (1-2-3).
    • The user may enter widget title (4).
    • In the widget by default are displayed all tasks in the current project with their assignee and due date (5).
    • On the top of the widget there is "Create New Task" button which is active (6).

 

  • The user changes the default settings of the widget:
    • The user selects "Manage properties" on the widget (1-2).



  • On the first tab "Objects to be displayed" (1) the user selects to see only tasks created from selected date on (here - December/23/2014) (1-2-3)  and clicks tab "Object details to be displayed" (4).

  • On the second tab "Object details to be displayed" the user selects (adds/ removes) task details to be displayed in the widget - here adds Started by and Priority of the tasks (1-2) and clicks "OK" (3) confirming the widget settings.

  • The system displays the task created after the selected date with the selected properties.

Changing the default settings for new tasks
  • The user has the option to change the default settings for creating new tasks (1-2-3-4).

  • The selected values appear in the corresponding fields when a new task is created from the widget.
    • The values may be changed also from the opened pop up screen "Create Task".

 

Create new task from the widget in the current (selected) context.

  • The user selects the action "Create New Task" from the widget.

  • A pop-up screen "Create Task" opens:
    • The Context is set to "Current Project". The user could select the context "Current Case" (or other context in the system) from the drop-down list for Context (1).
    • The Task type is set to the first task type from the drop-down list (2). The user could select different type of task.
      • The task details of the task are loaded and the user could fill in the necessary details (3-4-5). The task assignee by default is the current user and could be changed (5).
    •  The user saves the task by clicking the button "OK" (6).

  • The new task is created in the current (selected) context  and appears in the widget.
