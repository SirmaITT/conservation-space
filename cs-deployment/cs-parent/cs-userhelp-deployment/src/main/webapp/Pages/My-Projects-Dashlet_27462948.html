Sirma Platform : My Projects Dashlet

"My Projects" dashlet is displayed only on the Personal Dashboard and shows all projects to which the user is assigned as a resource. 

The projects in the dashlet are listed alphabetically by Project Name. 

  •  For each project in the dashlet are displayed:
    • Project ID, Project Type (optional), Project Name.  They all are a link (1) to open the corresponding project dashboard.
    •  A tool tip (2) displays additional project information: Owner, Status, Created on.


  • In the header is displayed the number of projects in the dashlet and the total number of all projects of the same type to which the user has access (1).
    • The following project filters, selectable from the drop down list (2) are applied on the dashlet:
      • Active projects (default) - this is the default filter and when selected the user sees all projects in Active state (Submitted, In progress, Approved, Suspended) for which he/ she is an owner or assignee.
      • Owned by me projects - shows only active projects for which the user is an owner.
      • I’m assigned to projects - shows only active projects in which the user is an assignee.
      • High Priority projects - shows active projects in "High" priority for which the user is an owner of the project or assignee to the  project.
      • Overdue Projects - shows active projects for which the "project end date" has passed (is before the current date)  and for which the user is an owner of the project or assignee to the  project.
      • My Favorites - shows all the items that the user has marked as favorites (only on My dashboard).
    • The items in the dashlet may be sorted by the following criteria, selectable from the drop down list (3):
      • Date modified on - this is the default sort criteria, descending order (the items recently created/ updated are on the top)

      • Title

      • Priority

      • Planned Start date

      • Planned End date

      • Status

    • For each combination of criteria the user is able to sort the items in the dashlet in ascending or descending order (4).
    • If the user wants to navigate to a project that's not in this dashlet he could search for it from Browse projects (5) link from the top lane of the dashlet.
    • From the dashlet the user could create a new project  selecting the action "New Project" from the "Actions" drop down menu (6).
    • Next to each project details in the dashlet there is a list with available actions on the project (7). The actions are restricted by user's role in the project.

 

    • The actions in general include: Edit Project details,   Project Members, Delete Project,  Approve project, Create Case, Create Task, Create Object, Suspend/ Restart Project,  Stop Project, Comment etc   (1-2).
