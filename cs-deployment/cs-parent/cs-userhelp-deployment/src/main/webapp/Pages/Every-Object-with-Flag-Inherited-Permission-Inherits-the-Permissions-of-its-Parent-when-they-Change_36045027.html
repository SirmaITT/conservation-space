Sirma Platform : Every Object with Flag Inherited Permission Inherits the Permissions of its Parent when they Change

Every very object with flag "Inherited permissions" or "Inherited and Special permissions" inherits the permissions of its parent when they change.

  • Inherited permissions of children are updated when update project permissions
  • Special permissions of children are not updated when update parent permissions

Inherited permissions of children are updated when update project permissions

  • A project is created with default permissions. The creator is Manager (1) and all other users are Consumers (2).

  • The case created in the project inherits (1) the permissions set on the project (2-3).

  • When the manager of the project edits the permissions of the project ( for example sets no permissions to all other users).

  • The permissions of the case automatically change (all other users are set with no permissions) and inherit the permissions set on the project.

Special permissions of children are not updated when update parent permissions

  • A project is created with default permissions. The creator is Manager (1) and all other users are Consumers (2).

  • A case is created (1) in the project with Inherited and Special permissions (2). The special permissions are displayed in the corresponding section (3).

  • When the manager of the project edits the permissions of the project (for example sets no permissions to all other users).

  • Then automatically the permissions of the case are updated as follows:
    • The Special permissions on the case (1) are not changed.
    • The Inherited permissions on the case (2) are updated and are as the permissions of the project (no permissions set to all other users).
