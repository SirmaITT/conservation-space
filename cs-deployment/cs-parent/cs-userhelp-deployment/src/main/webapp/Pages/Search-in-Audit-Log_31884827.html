Sirma Platform : Search in Audit Log

The administrator is able to search in the Audit Log by different criteria using a similar to the Advanced Search form.

  • Rules:
    • Example 1: Search by criteria
    • Example 2: Display the whole log
  • All types of contexts are stored in AL

Rules:

  • Search is case insensitive for all search criteria.
  • Most fields have auto suggest - when the user clicks in the field and types in text, the list with suggested values is being filtered on each keystroke.
  • Most fields allow selecting more than one value to be selected. The values are connected with "OR" operator.
  • For the "Event date" field a built-in calendar is implemented, allowing the user to select date and hour of the event.
  • The Context field displays the entities from the current state of the semantic database. This means that if a case's title is changed, the previous title will not appear in the Context field. However, previously recorded entries for the case will remain unchanged and they may be found only by other criteria.

Example 1: Search by criteria

  • The user enters the search criteria:
    •  selects "Event Date" (1-2-3-4-5).

    • If needed the user may open the Search Query field (1) to see the automatically generated query (2).
    • The user selects criteria "Object Type" (3) to be "Tasks" and "Projects" (4-5-6).

  • The user searches in the AL.
    • When the user finishes with entering data (1), the query string correspondent to the entered criteria is displayed (2).
    • The user clicks "Search" (3).
    • The record/s in the audit log corresponding to the entered criteria are displayed in the table "Found activities"(4).
    • The results are sorted by "Event date", the newest events on the top.
    • The results are paginated and the user may define the number of results to be displayed in a page (5). Available values are: 10, 25, 50,100.

    • In the search form in AL and in the results table Context is displayed with the Breadcrumb header (6) and the value(s) for Object are displayed with the Compact headers.

      • There are tool-tips with more information for the objects (6-1).

    • Each column in the table represents one criteria and the data in the table may be sorted by some of the columns ascending/ descending(7).
    • The user may select which columns to be added/ removed from the table selecting the correspondent check box from the list (7-8-9).

    • Paging is available at the bottom of the page and the user may browse the table (10).
    • There is information of the total number of records found from the search (11).

Example 2: Display the whole log

  • If no data is entered for the search criteria and the user clicks the "Search" (1) button then the whole log of the system would be visualized paginated in the table (2).

All types of contexts are stored in AL

In the Audit log all types of context are stored: Project (1), Case (2), Workflow (3), Task (4), Object (5), Document (6).
