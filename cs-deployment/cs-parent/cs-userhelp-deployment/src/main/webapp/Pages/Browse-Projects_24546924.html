Sirma Platform : Browse Projects

During browsing for existing project the user has a simple search functionality by keyword and sort functionality. Detailed search will be described in another topic (Search).

Constraints

Projects for which the visibility is set to "Visible only for project members" are visible for browsing only by users who are members for the respective project or users with the role of Administrator.

  • The user starts the Browse Projects operation from:
    • Personal Dashboard/ Projects/ Browse projects (1-2-3) or
    • Personal Dashboard/ My projects dashlet/ "Browse projects" (1-4) 

  • The system opens a screen where all projects that are visible for the current user are listed:
    • Projects for which the user is a member.
    • Projects for which is set up the value "Visible for all users".

Projects for which the visibility is set to "Visible only for project members" and the current user is not their member are not displayed in the list with projects, and the respective user will not know for their existence.

The projects in "Browse all projects" screen, are sorted by default by date of creation. The newest projects are on the top. The list is paginated.

For each project in the "Browse all projects" screen are visible the following details: Project ID, Project type, Project title, Project owner, Project status, Date of creation.

  • There is a search field on the top of the screen with the listed projects, where the user could enter a keyword (1), could select item from the auto suggest list (2) and search for  the project pressing the button Search (3). The search is case insensitive.

  • The system displays the search result where are listed only the projects, in which the entered keyword exists (1).
    • If the user wants to see again all projects, he/she selects the button List All (2).

  • In the list of projects, the Project ID, Project type, Project title, Project status are represented as a link. The user clicks on a link of a selected project.

  • The system opens the respective project dashboard.
