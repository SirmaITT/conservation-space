Sirma Platform : Add Saved Filter into a Library

The user is able to add custom filters (saved searches) in the project library and use it to organize better the content in the project. One and the same saved filter could be added to more than one projects.

When a filter saved in the library is selected all objects that satisfy the filtering criteria (the search query) are displayed in the search results. 

If the saved filter is created to search in the context of  "Current project", then when it is saved in the library of a specific project it is searching only documents in its context.

The filter visibility in the library depends on its own permissions, not on the project permissions.

  • The user opens the project library from the project dashboard (1-2).


  • The object library opens (1).
  • The user opens the Saved Filters drop-down menu (2) and selects the filter from the list of existing filters which should be added in the project (3-4).

  • The selected filter is loaded (1) and executed.
    • The user could view the filter details and see the saved search criteria, the description, etc (2-3-4).
    •  The results from the search are displayed (5).
  • The user selects the action "Add to Project" (6).

Notes:

  1. The action "Add to project" is active only when an existing filter is selected and loaded from the list with custom filters.
  2. If the same filter is already added to the project, then the "Add to project" action is inactive.

  • The filter is displayed in the library (1) in the leftmost section with its name (2).

  • Has Attachment/  Attached to relationship is automatically created between the project and the filter. The relation is visualized in the relationships tab in the project (1-2).
    • The action is logged in the audit log.

 

  • When a filter is selected from the library (1), then it is loaded in the search screen, automatically is executed and the search result, satisfying the query (2) is displayed to the user.
    • The context in which the custom defined filters are searching is not restricted to the current project.
