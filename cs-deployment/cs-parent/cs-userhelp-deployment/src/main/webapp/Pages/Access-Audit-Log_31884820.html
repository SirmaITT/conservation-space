Sirma Platform : Access Audit Log

Only members of the administrators group have access to the EMF audit log.

  • The user opens the audit log from EMF main menu Admin Tools/ Audit Log (1-2). Users that are not in the administrators group don't have access to the administration menu - the menu is hidden.

  • The system opens the audit log screen with the following sections:
    • Search criteria (1) - the user is able to remove (2) or add (3) different criteria selecting from the correspondent drop down menu (6):
      • Action (default visible)
      • User (default visible) 
      • Event date (from/ to - inclusive), date and time (default visible)
      • Object type (default visible)
      • Object title (default visible)
      • Available are also: Action (ID), Username, Context, Object sub-type, Object sub-type (ID), Object type (ID), Object state, Object state (ID), Previous state, Previous state (ID) etc.

The Context may be Document library, Object library, a specific case or a specific project; multiple values can be selected

Complex searches may be performed in the audit log. Search criteria may be specified with AND, OR and combinations with AND and OR (4).

Opening and closing brackets (5) may be used to in order to exclude some entries from the results.

For each search criteria the user is able to select from list (7) and specify how the entered data in field (8) to be treated. The user has the option to exclude some entries from the results (NOT)

    • Query (9) - The entered data in the search criteria section is transformed in a query string automatically. The user is able to create/ edit the data in the string manually.
    • Found activities  - After a search is performed (10), the results are displayed in the table (11).
