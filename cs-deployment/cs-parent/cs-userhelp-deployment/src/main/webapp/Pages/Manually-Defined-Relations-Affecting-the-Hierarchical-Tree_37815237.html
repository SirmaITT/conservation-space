Sirma Platform : Manually Defined Relations Affecting the Hierarchical Tree

The user has the opportunity to define custom relations between some or any of the enterprise objects. Each of these relations must be a part of the semantic ontology.

These types of relations allow for changes in the hierarchical tree (or browser) of the project that reflect the dependencies between documents and/or objects.

  • Relation "Is related to"
  • Relation "Depends on"
  • Relation "Involved in"

Relation "Is related to"

Example:

The user may want to state that a document is related to a selected enterprise object.

  • The user opens the document, selects tab Relationships (1), clicks the button Add relation (2), selects from the drop down menu the Relation type to be "Is related to" (3) and clicks "Select objects" (4) to manually create the relation to a selected object.

  • The system opens the Object picker and the user selects the object to which the relation to be created (1-2-3-4).

  • The relation is created between the document and the selected object

  •  The relation is visible in the object' Relationships widget.

  • The user opens the project dashboard and in the browser tree the related items are displayed (1-2).

Relation "Depends on"

Creating a relation of type Depends on between a case and a project (there is a Depends on relation defined in the test domain model).

Relation "Involved in"

Creating a relation of type Involved in between two objects (there is an Involved in relation defined in the test domain model).
