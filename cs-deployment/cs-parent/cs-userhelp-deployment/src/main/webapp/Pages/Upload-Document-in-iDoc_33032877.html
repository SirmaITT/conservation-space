Upload Document

New documents could be uploaded both in Edit or in Preview mode of an iDoc/ object.

The user could upload new documents from Actions/ Upload document in the landing page of an iDoc.

  • The user selects the action "Upload document" from the actions drop-down menu (1-2).

    • If the iDoc was not saved for the first time the system displays message and user is advised to save the iDoc first and then to upload documents in it.
    • the user should click "Cancel" and would be transferred to the landing page of the iDoc.

  • The pop-up screen for selecting documents opens (1) where the user:
    • Clicks the button "+Select File(s)" (2) .
    • Selects in the opened form (3) the necessary files for upload (4-5) from his/ her PC. The user click "Open" (6)
  • The selected files appear in the form for upload (7).

  • In the form for upload:
    • For each document there is a button Upload (1) and a button Remove(2). If selected the action applies only for the corresponding file.
    • There are buttons Remove All (3) and Upload All (6). If used the action applies for all selected files in the form.
    • For each document the user selects the Document Type from drop-down list (4), and enter a description of the document (5)
      • If needed the user fills in the mandatory data properties for the selected document type of the uploaded document.
    • If the user selects the button "Cancel" (7), this would cancel the upload of the selected documents.
  • The user clicks "Upload All" (6).

  • The system displays message "Upload Complete"for the successfully uploaded items.
  • The user clicks "OK".

  • The selected documents are uploaded in the iDoc.

Upload Document with Mandatory Data Properties

Some document types may be configured with mandatory data properties. Values for these properties should be entered before the n of the document.

The mandatory data properties could be selected from existing code lists or to be simple text values.  

Example:

  • The user selects the action "Upload" in the official section of selected case or in the landing page of an iDoc.
  • The system opens the "Upload Document" picker (1). The user selects the document (2) and should select the document type from the drop down menu (3). 
  • The system opens the form in which for the selected type ("Document for testing(2)") there are fields for mandatory properties:
    • Description (4) is not a mandatory field.
    • Test Field 1 (5) is a mandatory field - free text may be entered.
    • Test Field 2 (6) is a mandatory field - free text may be entered.
    • Test Field 3 (7) is a mandatory field - free text may be entered.
    • Created on (8) is a mandatory field. The automatically entered date of creation cannot be edited by the user.
    • Test Codelist (9) is a mandatory field. The user should select a value from the corresponding drop down list.
  • If the user selects the button Upload All (10) without entering data for the mandatory fields (here 5,6,7) the system displays error ( empty fields are in red) and the document from the selected type cannot be uploaded until data is entered for all mandatory fields.

  • The action is recorded in the Audit Log
