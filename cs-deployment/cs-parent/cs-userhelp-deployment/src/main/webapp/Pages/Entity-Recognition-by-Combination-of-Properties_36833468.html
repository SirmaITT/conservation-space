Sirma Platform : Entity Recognition by Combination of Properties

The system provides "entity recognition" functionality in uploaded and created documents, recognizing object by combination of some of its properties. Each property is non-unique, but becomes unique when combined with another property.

The user is able to specify (in the definition):

  1. The objects that should be recognized in the documents and/or the objects.
  2. The documents that should be recognized in the documents and/or the objects.
  3. The combination of properties for each object or document that should be searched.
  4. Where in the documents the system should search for the respective objects (title, description, content, etc.)

Example 1: This functionality  is needed because in a document there could be the name of a user <FirstName><space><FamilyName>, and neither the First Name is unique property, nor the Family Name, but the combination of the 2 names is unique (we don't have in the document the userID, which is unique). So the user is identified by the string which is a combination of the two name properties and a space symbol in between and the document and the user are linked.

Example 2: The part number and part revision of objects. There are many objects which have one and the same part number, but only the combination of part number and part revision makes the object unique. So when we see the string 123456D we know that this is the part with part number 123456 and the revision of the part is D. The same should do the entity recognition functionality - when the string 123456D is met in a document, the system should uniquely identify  part number 123456, revision D and should link the document with that specific part revision.

Rules:

  1. The user is able to set up the system (in the definitions) and define the combination of properties that should serve for unique identification of an object in a document
  2. The user is able to set up the sequence of the properties in a string and the separator between the properties: <prop1><prop2>, or <prop1> <prop2>, or <prop1>/<prop2>_<prop3>, etc.
