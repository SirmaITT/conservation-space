Sirma Platform : Move Document and Move Document to Other Section

The user is able to move a document in the system in order to change its parent. The action may be performed on uploaded/ created documents and as a result the parent to different domain object/ document/ case / project is changed.

Rules:

  1. When the user selects Move action on a document then the object picker is displayed.
  2. By default in the object picker, in field "Object Type" is displayed the type of the current parent of the document. For example if a document is created in case and the user selects the  "Move" action on the document, then by default the Case selected as a target object is displayed.  If a document is created/ uploaded in a Painting, then by default in the object picker Painting as a target object is displayed.
  3. In the Object picker the user can change the default object type to any object type, which could have child document ( this means every object except for task and workflow) - Project, Case, Domain specific object, Document.
  4. In the object picker the user can search by the sub-types of the selected object type and may change the context.
  5. If a case is selected as a target object - then the user can select the section in the case, where to move the document.
  6. Revisions of a document could not be moved alone.
  7. A document is moved together with all its children documents.
  8. Objects for which the user does not have permissions to create content (he is a consumer, or with no permissions) are not displayed in the search result in the picker.
  9. If the move operation is successful, then a message is displayed "The document is moved successfully to the selected target. The permissions of the document are changed. You can review them in the Permissions section of the document."
  10. When a document is moved to new parent object, then the "Part of/ Has child" relationship with the old parent is deleted, and "Part of/ Has child" relationship is created with the new parent.
  11. The existing relationships (except part of/ has child) of the moved document are not changed with the movement
  12. The move action is recorded in the audit log.
  13. In the context field of the action in the audit log there are two locations/ parents - the source location and the target location.

  • Move Document to Other Section

Move Document

  • The user starts the action from:
    • Personal Dashboard/ My Content dashlet/ Actions/ Move Document for a selected document.
    • Case Dashboard/ Content dashlet/ Actions/ Move Document for a selected document.
    • Case Dashboard/ tab Documents and Media/ Actions/ Move Document for a selected document (1-2-3)
    • Document landing page/ Actions/ Move Document

  • The system opens the object picker for selecting the target destination for the moved document (1).
    • By default in the object picker, in field "Object Type" is displayed the type of the current parent of the document (2). A search is automatically performed and results are listed (3).
    • The user may select any of the items from the displayed results (4-5). If a case is selected as a target object - then the user should select the section in the case, where to move the document.
    • The user may change the default object type to any object type, which could have child document, may search by the sub-types of the selected object type and may change the context.

  • The user changes the default object type (1), enters other search criteria (2), performs search (3), selects the target object from the search results and confirms the operation (5).

  • The system displays notification message for  the successful operation (1). The user clicks "Yes" (2).

  • The document is moved in the selected target location ( here - on the project level of the selected project) (1-2-3).

  • The action is logged in the Audit Log.
    • In the Context field of the record the source location and the target location of the moved document are displayed.

Move Document to Other Section

The action is performed when the user wants to move a document from one section of a case in another section of the same case.

The action cannot be performed for documents which are not in a case section.

  • The user starts the action from:
    • Personal Dashboard/ My Content dashlet/ Actions/ Move Document to other case for a selected document which is in a case section.
    • Case Dashboard/ Content dashlet/ Actions/ Move Document to other case for a selected document which is in a case section.
    • Case Dashboard/ tab Documents and Media/ Actions/ Move Document to other case for a selected document in a case section (1-2-3-4).
    • Document preview/ Actions/ Move Document to other case if the document is in a case section.

  • The system opens the Section picker (1) with the list of all sections in this case. The user selects the target section (2) where the document should be moved and clicks the button "Yes" (3).

Note: If the user selects to move the document in the same section an error message is displayed and the action is blocked.

  • The document is moved in the selected section (1-2) of the same case.
    • The document is not visible in the section from where the operation was started.

  • The action is recorded in the audit log
