Sirma Platform : Default Permissions on Created Library

When a new domain (object) library is set up in the system, it is created initially with consumer permissions for all system users. The Administrator is with manager permissions and can easily edit (add/ restrict) permissions given to users and/ or groups on all objects in the library and their sub-objects (attached documents, images, iDocs).

  • View default permissions on a Library
  • Existing objects inherit the default library permissions

View default permissions on a Library

  • The user opens an empty library (with set default permissions) from the Objects library menu (1-2) .

  • The system opens the corresponding library (1). Automatically a basic search is executed, showing no object of this object type found (2-3).
  • The user selects to open the Library Details from the actions menu (4-5).

  • The system opens the corresponding object library landing page (1).
  • The user opens the "Permissions" tab (2-3) for the object (here - Mark) library landing page.
    • The default permissions set on the empty object library are:
      • Administrator (admin) - MANAGER permissions (4)
      • Administrators Group  - MANAGER permissions (5)
      • All other users - CONSUMER permissions (6)
  • The action Edit Permissions(7) is visible and active only for users with MANAGER permissions.
     

Existing objects inherit the default library permissions

  • The user opens the object  landing page of a created object in a library with set default library permissions and selects tab Permissions (1). 
    • The permissions flag Permission type for the object is set to Inherited (2).
    • The object created in a library with set default library permissions inherits the permissions set on the library (3).
