Sirma Platform : End of workflow notification

When the  "Send notifications to task assignees" check-box is selected (default) in the start task of the workflow, all users assigned to tasks in the workflow, are notified when the workflow is completed.

Rules:

  1. In the start task of the workflow there is a check-box "Send notifications to task assignees". If the check-box is selected then e-mails are sent to all users, based on the workflow settings.
  2. The above check-box is selected by default.
  3. E-mail is sent to every user who has been assigned to a task in the workflow and completes the respective task (if the user has been an assignee of the the task, but the task was reassigned and completed by another user, only the second user receives an e-mail).
  4. Different emails (as content and template) are sent based on the transition selected on the completion of the workflow.
  5. For every type of workflow different content of the notification e-mails could be specified.
  6. The e-mail template may be set to display the draft of the document/ domain object, or the published revision.
  7. If two or more documents/ objects are processed in the workflow  the template may be set to display the links to all of them.

Example template of an e-mail:

Subject: Released ECN <Title of approved revision of document of type ECN, published in the WF>

Body:
ECN <Title of approved revision of document of type ECN, published in the WF> has been released.
You can view the document through this link <here the link that points to the document in SEIP is displayed>

The following parts have been released also:

Part <The Part Number of the approved revision of the part published in the WF> link

Part <....>

Part <...>
