Sirma Platform : Define New Codelist

Only users with administrator permissions have the option for defining new codelist.

  • The user selects the action  "New Code list" in the main menu of the codelist management panel.

  • The system displays a blank form for entering codelist parameters in editable mode (1).
    • The current date is set by default for "Valid from" (2). The value could be edited.
    • Mandatory fields are marked (3-4).
    • The button "Save" (5) is not active because not all mandatory fields are filled.

  • The user fills in the mandatory fields and/or data for other parameters of the codelist (1-2-3-4-5).

Constraints, applied on the form fields, are:

    1. the code (identifier) of the codelist must be unique for the system;
    2. the code (identifier) of the codelist cannot contain spaces, special characters, and letters; consists only of digits.
    3. if validity start date and validity end date are present, then the validity end date must be later than the validity start date and must be later than the current date;
    4. description in English is a mandatory field
    5. the masterCL is a drop down field containing all valid (currently and in the future) codelists for the tenant

    6. the "Display type" field is a drop down field with options: Value, Description, Value and description

    7. the "Sort by" field is a drop down field with options: Value ascending, Value descending, Description ascending, Description descending, Order ascending, Order descending

  • The user clicks "Save" (6).

  • The system displays a confirmation dialog form (1).
    • If the administrator confirms the operation (2) the system creates a new codelist with no code values in it and displays the form for codelist details in preview mode.
    • If the administrator does not confirm the operation (3) the system displays a confirmation dialog form:

      • If the administrator confirms the cancel operation - the system reverts all changes and the new code list form is closed.

      • If the administrator does not confirm the operation - the system returns to the new code list form with latest changes made.

The administrator confirms the operation (2).

 

  • The system creates a new Codelist (1) with no Code values (2) in it and displays the form for codelist details in preview mode.
