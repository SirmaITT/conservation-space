Sirma Platform : Create Document Directly in Project

The user is able to create documents directly in the project.These documents are created as part of the project, not as part of its cases.

Rules:

  1. In the project there is action “Create document” in the list with all actions on project level.
  2. Only document types valid for the respective project and its cases could be created in the project.
  3. Part of/ has child relationship is created between the document and the project.
  • The user opens the project dashboard and selects the action “Create document”  from the list with all actions on project level (1-2-3).

  • The system opens the form for document creation (1).
  • The user selects from the drop-down list of allowed document types for this type of project and its cases (2-3).

  • The selected document type (1) appears in the corresponding field. The user may enter a description (2).
    • If needed the user fills in the mandatory data properties for the selected document type of the uploaded document.
  • The user clicks "OK" (3).

  • The system opens the empty form for creation of document of the selected type. The user enters the title (1) adds widgets and edits the document (2-3-4). 

  • In the saved document in the Relationships widget appears the automatically created relation of type "Part of" between the created document and the project (1).
  • The user opens the tab "Relationships" (2).

  • The automatically created relation between the project and the document "Has child" is displayed.

  • The created document appears in the Content dashlet on the project's dashboard (1-2).

  • The document is visible in the library of the project (1-2).
