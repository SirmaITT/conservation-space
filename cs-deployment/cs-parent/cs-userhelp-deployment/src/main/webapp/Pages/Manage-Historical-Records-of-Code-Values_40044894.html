Sirma Platform : Manage Historical Records of Code Values

A historical version of the code value is created when the code value is edited (data is modified).

A historical record means that there is a record for the code value that is/will become/was valid after the first creation of the code value. The validity period of the historical record is followed by the validity period of another record of the codelist/code value.

The historical record may be invalid or may be valid currently.

  • The user has edited the code value (INIT) in Code list "1".
  • The user searches for code values beginning with "ini*" in the codelist (1-2).
  • The system displays the results (3).
    • a historical record of the code value was created (4) after the edit action.
    • the new edited code value (5) is displayed in the list.

Note:  Records that are valid in the future are modified without creating historical records.
