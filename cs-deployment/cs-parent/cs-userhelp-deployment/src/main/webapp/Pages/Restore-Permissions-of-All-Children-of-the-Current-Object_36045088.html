Sirma Platform : Restore Permissions of All Children of the Current Object

A user with the role of Administrator or Manager in a project is able to easily restore the permissions of all objects in given hierarchy.  With this action the user actually easily removes all the special permissions and activates the inherited permissions on objects.

Rules:

  1. There is "Restore children permissions" action for every project and for every object part of a project that has children.
  2. The action is visible only when Edit permissions is selected for the project or for the respective object.
  3. Only an user with management role can restore permissions.

Example:

  • In a project the permissions are set as follows: admin admin is with Manager permissions and System_All_Other_Users are with Consumer permissions.

  • In the Project there is a case "1CS" where Special permissions of  Collaborator are given to an user (Adrian adr) (1-2-3)

  • In the Project there is a case "2CS" where Special permissions of  Collaborator are given to two users (Anna Radeva and Dimitar Glezev) (1-2-3)

  • The manager of the project selects the button "Edit Permissions" in the project details page (1).
  • The button "Restore children permissions" is activated (2) and the manager selects it.

  • The system displays the message "All special permissions of the children objects will be removed! The action is non-reversible! Are you sure that you want to continue?"
  • The user selects the button "OK".

  • All permissions of the children objects of the project are set to Inherited, and the special permissions are removed (1-2-3).
    • In case "1CS"

    • In case "2CS"

  • The action Edit Permissions for all objects with restored permissions is logged in the audit log .
    • The permissions of the project or of the parent object, from which is selected "Restore children permissions" are not changed.

Note: Edit permissions is performed only for objects with "Special" or "Inherited and Special" permissions. For objects with only Inherited permissions no changes are made.
