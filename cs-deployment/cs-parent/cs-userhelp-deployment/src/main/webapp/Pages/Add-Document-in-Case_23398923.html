Sirma Platform : Add Document in Case

Attached (added) documents are these documents that are not defined previously as mandatory documents for the selected type of case and only the user chooses what kind of documents are needed for that case.

Preconditions

1. The actions Add, Delete, Remove document  in a case may be carried out only if:
       - the case is in state "New" and "Create New Case" is in progress or
       - the case is in an active state and "Edit Case Details" is completed successfully
2. Attachment of documents with identical names in one case is not allowed.

Start Action Add Document in Case

The user may add as many documents to the case as needed.

  • The user starts the operation by clicking tab Documents from:
    • Case Details/ tab Documents
    • Case Dashboard/ tab Documents(1-2)

Add Document

  • The system opens the screen with the defined sections in the case.  The user selects the drop down menu "Actions" (1) of the section where document/s would be added and selects an action from the list (2).

Upload Document
  • The user selects the action "Upload" from the drop down list to add document/s from his/ her PC.
    • The system opens the "Upload Document" picker (1), the user clicks "Select File(s)" (2).
    • The system opens the "File Upload" screen (3) and the user searches and selects document from his/ her local PC (4-5-6).

  • The selected document is displayed in the list (1). 
    • The user has options to remove a document (2) or all documents from the list (3).
    • The user may enter a description in plain text for the document (4).
    • The user select  from a drop down menu the Document Type which will be displayed for that document in the case (5).
      • If needed the user fills in the mandatory data properties for the selected document type of the uploaded document.
    • When the user is ready he/ she clicks "Upload Document" (6).

  • The selected file (1) is uploaded in the section of the case from where the operation was started.

    • For every attached document the following information is displayed:

      • Thumbnail of the document
      • Document Type
      • Name of the attached document
      • Mime type  of the attached document
      • Document version
      • Last modified by - the user who attached/ modified the document
      • Modified on - Date and Time of the attachment/ modifi
    • Next to the document there is a drop down menu which displays all possible actions that the current user may perform with the corresponding document (2-3).

  • The Document type and Name are a direct link to open the document in preview mode.

  • The document appears also on the Case Dashboard: in the Browser, in the Documents dashlet and in the Media dashlet (if the document is an image of type jpg, tiff, png, gif, bmp).

  • If during the process of adding the document/s in the case another document with the same name is found, the system automatically renames the document and attaches "_1", "_2" ... after the name, depending on the number of attached documents to the case with the same name.
Create Document

The action is described in the topic in Intelligent Document Management/Create new iDoc.

Add Document
  • The user selects the tab "Documents" in the Case Dashboard or Case Details and chooses the action "Add Document" for any of the documents sections in the case in order to add document/s in the case from other cases or from the whole system.
  • The Document picker screen (1) opens, with Object Type field restricted to objects of type "Document" and "Image" (later with audio and video)
    • The user is able to search/ browse (2) the system and select one or more documents (3-4-5-6-7).

    • The selected documents are displayed in the section from which the attachment is initiated.

Relations

    • Automatically relations of type "Part of/ has child" are created between the project/ case and the added document/s (1-2).
