Sirma Platform : Create e-mail Distribution List in a Workflow

In a workflow the user is able to create a list of users and/or groups of users, who should receive e-mails at the completion of the workflow, or at defined task transition.

The  e-mail distribution list may be updated in every task of the workflow (from the start task to the final task).

Rules:

  1. The user could configure if a workflow should support e-mail distribution list or not.
  2. There is a section e-mail distribution list (Watchers) in each task of a workflow, which is configured to support e-mail distributions.
  3. The list is configurable and may contain just the users and/or groups that can be set in the definition of each task of the workflow.
  4. In the section e-mail distribution there is a user picker in which the user could search and select users and groups of users.
  5. The users added in one task in the e-mail distribution list are visible in the next task of the workflow.
  6. Every user executing a task from the workflow could add/ remove users from the e-mail distribution list.
  7. When "Send notifications to task assignees" check-box is checked, then every user who is assigned to a task in the workflow is automatically added in the e-mail distribution list (even if the task is reassigned later)
  8. Every user from the e-mail distribution list receives an e-mail in predefined template at the completion of the workflow.

Example:
  • When the user starts a workflow or opens a task assigned to him/ her:
    • He checks the check-box "Send notifications to task assignees" (1) and is automatically added in the e-mail distribution list.
    • He clicks the button "Select" (2) to add/ remove other users/ groups in the e-mail distribution list.

  • The system opens the picker from  where the user could search and select users and groups of users (1-2) and add them in the e-mail distribution list.
    • Entries in the list could be removed by clicking the corresponding  button "-" (3).
  • When ready the user clicks the button "Save"(4).

  • The selected user/ group appears in the list and will receive notification emails for the completion of the workflow.
