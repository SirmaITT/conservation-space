Sirma Platform : Upload New Version

Preconditions

  1. The action is not performed for iDocs.
  2. When a new version of the document is uploaded the extension of the new version of document must be the same as the extension of the first version of the document.
  3. For a document which is locked for edit, the action "Upload New Version" is active only for the user who has locked the document for edit and is not active for other users (except for users with administrative permissions).
  • The user starts the action from:
    • Personal Dashboard/ My Documents dashlet/ Actions/ Upload New Version for a selected document
    • Case Dashboard/ Documents dashlet/ Actions/ Upload New Version for a selected document
    • Case Dashboard/ tab Documents/ Actions/ Upload New Version for a selected document 
    • Document preview/ Actions/ Upload New Version (1-2)

  • The system opens the screen "Upload New Document Version" (1) where the user selects whether the changes in the new version are "Minor" or "Major" (2). By default the "Minor" version is selected.
    • In case of "Minor" document changes - the minor version number is incremented with 0.1 (Example: v1.1 becomes v1.2)
    • In case of "Major" document changes - the major version number is incremented with 1.0 (Example: v1.1 becomes v2.0)

The user enters a Description (3) (mandatory) of the changes and clicks the button "Add file" (4) 

  • The system opens the screen for selecting a document from the local PC. The user selects the file (4-5).

  • The file appears in the screen for uploading a new version of the document (1). The selected document may be deleted (2) and a new one be selected for upload.
  • The user confirms the upload by clicking the button "Upload" (3).

  • The operation starts:

  • When the upload operation finishes, the system opens the document in preview mode (1):
    • A new version of the document is attached to the case:
      • the Title and the Description of the document are preserved as they were in the old version.
      • Next to the Title of document the new version number is recorded (2).
    • The new version of the document is saved in section Document History/ Last version (2). The description is visible under the name of the user who uploaded it (3).
    • The old version of the document is saved in section Document History/ Older versions . For the old versions different actions may be performed (4).
