Sirma Platform : Remove Saved Filter from Library

The user is able to remove custom saved filters (saved searches) from the project library.

Rules:

  1. There is action "Remove filter from library" for each filter that is attached to a project library. The action is in the Saved Filters section.
  2. The action "Remove filter from library" is active only for filters that are attached to a library.
  3. When the user selects "Remove filter from library" next to a saved filter, the filter is not visible in the library any more.
  4. Attached to/ has attachment relationship between the filter and the project is deleted.
  5. Only users with respective permissions on the project could remove filter from library (manager, collaborator)
  6. The action "Remove filter from library" is logged in the audit log.
  7. When the user clicks "Add filter to library", the filter must be added and remain selected in the search form.
  • In a project (1) with added saved filter in the library (2) the user opens the tab Relationships (3).
    • The relation of type Has Attachment between the project and the added saved filter in the library is visible (4).
  • The user opens the library of the project clicking the button All (5).

  • The system displays the content in the library (1). The user opens the section Saved Filters (2).
  • The user selects the added filter in the library (3).

  • The filter is loaded (1) and executed automatically. Results are displayed (2).Â 
  • There is a button "Remove filter from library". The user clicks the button (3).

  • The filter is removed from the project's library (1).
  • The user clicks the tab Relationships (2).

  • The relation with the saved filter is removed.

  • The action "Remove filter from library is recorded in the audit log.
