Sirma Platform : Activate/ Deactivate Inherited Permissions

The inherited permissions on an object are not editable. The user is able to deactivate all of them and to have only special permissions on the object, or to activate them back.


Rules:

  1. There is an action "Deactivate inherited permissions", for objects with flag "Inherited and Special permissions".
  2. There is an action "Activate inherited permissions", for objects with flag "Special permissions".
  3. There is no action "Deactivate inherited permissions", for objects with flag "Inherited permissions".

  • Deactivate Inherited permissions on enterprise object with existing Special permissions
  • Activate Inherited permissions for object with only Special permissions
  • Inherited permissions on enterprise object without existing Special permissions cannot be deactivated

Deactivate Inherited permissions on enterprise object with existing Special permissions

  • An user with edit permissions on object with Inherited and Special permissions (1-2-3) selects the action "Edit permissions" (4).

  • The permissions section opens in edit mode. In the Inherited Permissions section the check box Enable inherited permissions is active (can be selected/ deselected).
  • The user deselects the check box.

  • The check box Enable inherited permissions (1) is deselected. All users/ groups in the section Inherited Permissions are deactivated.
  • The user saves the changes (2).

  • The changes are saved. The permission type of the object changes to Special (1).
    • The Special Permissions on the object are not changed (2).
    • The section Inherited Permissions is empty (3) and the object is not visible to all other users.
    • The Management permissions are not changed.

Activate Inherited permissions for object with only Special permissions

  • An user with edit permissions on object with Special permissions (1) selects the action "Edit permissions".
  • The permissions section opens in edit mode. In the Inherited Permissions section the check box Enable inherited permissions is active (can be selected/ deselected).
  • The user selects the check box (2).

  • All inherited permissions on the object get restored to the users/groups (1).
  • The user saves the changes (2).

  • The Permission type of the object changes to Inherited and Special (1).
    • The Special Permissions on the object are not changed.
    • The object becomes visible to all users (2).
    • The Management permissions are not changed.

Inherited permissions on enterprise object without existing Special permissions cannot be deactivated

  • The user with edit permissions on an object with only Inherited permissions opens the Permissions section of the object in edit mode.
    • The Permission type is Inherited (1).
    • There are no users or groups with Special Permissions (2).
    • In the Inherited Permissions section the check box "Enable inherited permissions" is selected and inactive (3).

Note: For users without edit permissions on the object the action Edit Permissions is inactive and they cannot open the permissions section in edit mode.
