Sirma Platform : Set Additional Permissions to Projects Visible to All Users

Any user with the role of Manager in a project for projects flagged as "Visible to all users" is able to set special permissions to those users who are not members in the project. Currently all the users which are not members of the project are consumers by default.

  • Default permissions set on projects
    • Projects created with visibility set to "Visible for all users"
    • Projects created with visibility set to "Visible for project members only"
  • Set new role of all users, which are not members of the project

Default permissions set on projects

Projects created with visibility set to "Visible for all users"

  • Default roles set in Project members (2) section are:

    • For the creator of the project - MANAGER (3).

    • For all other users, which are not members of the project - CONSUMER (4).

Projects created with visibility set to "Visible for project members only"

  • Default roles set in Project members (2) section are:
    • For the creator of the project - MANAGER (3).
    • For all other users, which are not members of the project - NO_PERMISSION (4).

Set new role of all users, which are not members of the project

Example: In a project the manager of the project (with role "Manager") wants all system users to share new ideas and/ or problems and setsÂ  "All Other Users" to be with the role "Contributor" on the project.

  • The user with role MANAGER in the project selects tab Permissions (1) and clicks the button Edit Permissions (2).

  • The manager of the project selects the new role for all other users from the drop down list (1-2).

  • The new selected role appears in the corresponding field (1). The manager of the project clicks "Save Permissions" (2).

  • The changed permissions are displayed in the Permissions section (1-2).

  • The changed permissions are displayed in the Project members section (1-2).
