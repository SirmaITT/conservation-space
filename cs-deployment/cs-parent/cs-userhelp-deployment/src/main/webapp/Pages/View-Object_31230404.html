View Types

An object (like other primary system objects) has the following view types:

List view

This is the view of the object with a small icon and main object details. The view is used in the project/ case dashboards, in the objects dashlet.

In the list view the object is presented with:

  • Icon - with different icons (1) are represented all specific types of business domain objects (documents, tasks, cases, projects, etc.).
  • Object ID - the unique object ID
  • Object Name - the Title of the object.
  • Object Type

The ID and Name are presented as selectable links (2), which when clicked the object detailed view is opened.

Thumbnail view

This is the view of the object with a thumbnail in front of it and main object details. This view is used in the objects repository, in the search results, in the Objects tab (1) in the case.

In the thumbnail view the object is presented with:

  • Thumbnail (2) - this is the thumbnail of the image which is linked to the object with the relation type 'has Primary Image'
  • Object ID - the unique object ID
  • Object Name - the Title of the object.
  • Object Type
  • Other important metadata

The ID and Name of the object are presented as selectable links, which when clicked the object detailed view is opened.

 

Detailed view (Object Landing Page) 

When an object is selected (from the list view, or from the thumbnail view), the object landing page is opened where all object details are displayed.

The object landing page has the following sections:

  • Object Details (1) - here are visible all object details, as defined for the specific object type.
  • Actions (2) - in this section the user will see all actions he could perform on an object. The actions visibility depends on the user permissions.

    • The following actions could be performed on an object:(NOT allare READY): Edit Object, Delete Object, Manage Permissions (see Manage Permissions on Object Level section), Comment (see Object Discussions), Lock/ Unlock, Clone, Create Reference with other content  (see Manage References), View history version, Move to another section

  • Related Content (3) (See Manage References)

  • Comments (4) (see Object Discussions)

  • Cases (5) - in this section are listed all cases, where the object has been attached/ created/ selected  for manipulation.

    • For each case in the section there is information about: Case ID, Case Type, Case status, Last Modified on, Last modified by.

    • Case ID, Type and Status are presented as selectable links, and the user could click on them at any time to see the details on the performed work.

  • Version History (6)  - in this section are displayed all previous versions of the object.

    • For each version there is information about: (NOT READY)
      • short description of the change that has been done
    • Each existing history version of the object could be opened for review:
      • On the top of the Object Details section a message to the user is displayed - "This is a history version of an object. If you want to view the latest version click here.".
      • When the user clicks on word 'here', he/she would be transferred back to the current object version.

All sections are collapsed by default, the user could expand the one he/she needs

 

 

Tab "Objects"

  • In the Case dashboard  there is a tab Objects (1), where in thumbnail view all objects (2) associated with the case are listed.
  • Next to each object in the tab there is list with the actions that could be performed on it (3-4).
  • Based on the case definition:
    • the case could have one or many objects associated with it
    • the objects in the case could be grouped in different sections (5). For different case types different object sections could be set.


For example: There is a survey on 1000 objects from the same type (medals, coins, etc.) Only 20 would be selected for real examination, and the rest would just be included into the survey report.
In the Case/ Objects Tab  could be created 2 sections: 
  • 'Random Sampling Object List' - where will be listed only the object which have been selected as samples for examination
  •  'Objects for examination' - here will be listed all other objects. Thus the user would easily see the object groups.
