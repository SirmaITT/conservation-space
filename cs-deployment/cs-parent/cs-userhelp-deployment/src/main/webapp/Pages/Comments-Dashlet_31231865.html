Sirma Platform : Comments Dashlet

The dashlet is context dependent:

  1. On the Personal dashboard in the dashlet are displayed the comments created by the user or his replies to existing comments.
  2. On the Project dashlet are displayed the comments on objects associated with the project - the project itself, tasks, sub-projects, documents, etc.
  3. On the Case dashboard in the dashlet are displayed the comments in objects associated with the case/ sub project - documents, tasks, images, etc.
  • The comments in the dashlet could be filtered by the combination of the filters (1-2) Author, From/ To date, Category, Tags, Keyword, Status.  The filters show all comments created or to which a reply has been made.
    • Filter by Author - the user is able to select a name from all the users who have commented/ replied to comments, or All users (when empty)
    • Filter by period - From date/ To date
    • Filter by Category - the user is able to select from the list: Note to self, Editorial question/ suggestion, Request assistance, All (3-4)
    • Filter by Tags -
    • Filter by Keyword - filter comments by whole or part of a word/ phrase from its title
    • Filter by Status - the user is able to select from the list: Open, Closed, All (when empty)

When the user clicks "Filter" (5), the system would display the comments that satisfy the selected criteria.

  • For each comment (1) in the dashlet is displayed information about:
    • Title/Topic - the topic of the comment
    • Object ID - the ID of the object to which is made the comment
    • Created by
    • Created on
  • The comments maybe sorted by (2-3): Creation date or Last Comment date.
    There is a sign next to each comment showing the number of replies on a comment, and if clicked the section with the comment would be extended to view all replies.

  • When the user clicks on the topic of the comment (1), the comment section is extended to view the comment body and the replies on the comment (2).
  • The user may enter a new reply (3) or perform actions  (4) on the comment and the replies depending on his/ her permissions.

  • Next to each comment there is an icon representing the object to which the comment is made. The icon is a link to open the object's landing page.
