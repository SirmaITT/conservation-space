Sirma Platform : Print iDoc

Only the Document Content section form the whole iDoc page is submitted for printing. (The document tree view and the comments sections cannot be printed.)

  • The iDoc is opened in preview mode and the user selects the "Print" action from the drop down menu (1-2).
    The print action could also be active while documents are browsed in the system, and next to each document there would be a list with actions on it.

  • (Not Ready) The system opens the iDoc in Print Preview mode, which could be different than the document Preview mode.

In Print Preview mode the document is paginated and each page has header and footer, there could be also page breaks defined, if we want a sections to start on a new page, etc. (For more details see iDoc Content Management use cases).(Headers and Footers could be defined as widgets in the document content, but also some simple headers and footers could be defined in the Print Preview mode.)

While the document is opened in Print Preview mode, the user is able to:

  1. set up the document orientation - Portrait or Landscape
  2. scale the percent of the content to fit in one page (For example we may want to scale the document content to fit to a page - this would mean that font will me minimized till the whole document fits to 1 page.)
  3. change the dimensions of top, bottom, left and right margins of the document
  4. define simple header and footer
  • The user selects the printer on which the document to be printed (1-2-3).

  • The document content section of the iDoc is printed on the selected printer.
