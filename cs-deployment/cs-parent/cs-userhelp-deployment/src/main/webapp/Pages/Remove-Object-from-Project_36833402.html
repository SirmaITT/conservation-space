Sirma Platform : Remove Object from Project

Users with relevant permissions are able to remove (detach) objects from a project. The removed (detached) objects are not deleted but are only removed from the project.

Rules:

  1. There is "Remove Object" action on the object in the Objects dashlet .
  2. The action is active only in the context of a project and when a relationship of type Has Attachment/ Is Attached To exists between the project  and the object. (Note: one object could be attached to more than one projects, or not to be attached to any project)
  3. The action is active only when the object is explicitly attached to the project, but if it is attached to cases in the project it is not active.
  4. When the action is selected the relationship of type Has Attachment/ Is Attached To between the project and object is deleted.
  5. Only the users with Consumer role in the project cannot detach objects

Example:

  • There is an attached object on project level. Relationship of type Has Attachment/ Is Attached To exists between the project  and the object.

  • The user selects the action "Remove Object" for the  object in the Objects dashlet of the project (1-2-3-4).

  • The system displays confirmation message and the user confirms the operation.

  • The object is removed from the project.

  • The relationship of type Has Attachment/ Is Attached To between the project  and the object is deleted (1-2).

  • The action is recorded in the Audit Log.
