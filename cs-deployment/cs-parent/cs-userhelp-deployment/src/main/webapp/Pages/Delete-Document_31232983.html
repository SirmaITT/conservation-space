Sirma Platform : Delete Document

The action "Delete Document" is performed on added documents in project/ case and is available for users with corresponding permissions when the project/ case is NOT in a final state ("Completed"/ "Stopped").

An user, who is not the creator of a document attached to a case (the document is created in another case or project, from another user) is not able to delete it.

  • The user starts the action from :
    • Documents dashlet /drop down menu of actions for an added document.
    • page Documents/ drop down menu of actions for an added document in a selected section/ Remove Document (1-2-3).

  • A confirmation message is displayed. The user confirms the operation.
    • (Not Ready) The message contains information for the number of projects and cases in which this document is used.

  • The document is deleted and is no more displayed in the the case section.

  • The relation between the case and the document is deleted.

  • The action is recorded in the Audit Log.
  • After the operation "Delete Document" the document could NOT be found in the document library.
