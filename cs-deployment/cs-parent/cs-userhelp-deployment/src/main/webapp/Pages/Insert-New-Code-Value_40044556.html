Sirma Platform : Insert New Code Value

Only users with administrator permissions have the option for inserting a new code value in code lists.

The option for adding a new code value is available in the codelist preview form (with the list of all code values in the codelist).

The option for new code value is active only if the codelist is currently valid or its validity has not started yet.

  • The user opens a code list in preview mode (1). In the code list there is a table with code values (2).
  • The user clicks the button "Add new code value" (3) in the code value section.

  • The system displays a new empty form in editable mode so that the administrator may enter the data for the new code value in the code list (1). The user enters data for the new code value in all mandatory fields and or data in other fields (2-3-4-5-6).

Applied rules and constraints on the form:

  1. the code (identifier) of the code value must be unique for the codelist (2);
  2. supported languages are inherited from the codelist and tenants so all fields for description in a specific language(s), defined by the codelist and tenants, are mandatory (5-6).
  3. the validity start date is set by default to the current date and time is round up to next whole hour (eg. 09:00, 10:00, ..., 19:00, 20:00, etc.)
  4. the validity start date field is editable on the form (3)
  5. the validity end date field is editable on the form (4)
  6. the validity start date must be later than the current date and time plus the next whole hour
  7. the validity end date must be later than the current date and time plus the next whole hour
  8. the validity end date must be later than the validity start date
  9. the masterCV field may contains only one value
  10. the masterCV, if entered, must be a value from the masterCL of the codelist to which the current code value belongs

All other fields are editable with no assigned value by default.

Note: If there are validation errors on the form the Save button is not active and the new code value cannot be saved.

  • The user and clicks "Save" (7).

  • The system displays a confirmation dialog form (1).
    •  If the administrator confirms the operation (2), the system creates a new code value record within the codelist.
    • If the administrator does not confirm the operation (3), the system returns to the code value edit form with latest changes made.

The user clicks "Yes" (2).

  • The system creates a new code value in the selected code list (1-2-3).
