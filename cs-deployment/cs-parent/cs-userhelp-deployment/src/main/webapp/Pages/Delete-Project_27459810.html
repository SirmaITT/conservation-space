Delete project in "Submitted"

In general an user deletes a project only when he/she  has created it by mistake, for example of a wrong type. A project could be deleted only if in status "Submitted".  

Preconditions 

  1. Only the creator of a project, an user with the role of "Project Manager" for the project or an user with the role of "Administrator" could delete a project. 
  2. Only projects in state "Submitted" could be deleted. If there are documents, images, tasks, or other objects created in the project they would also be deleted. 
  •   The user who has the permissions to delete the project selects the button "Delete Project"  from:
    • Project dashboard/ "Delete Project" 
    • View Project Profile/ Actions/ "Delete Project" (1-2)
    • Personal dashboard/ My projects dashlet/ "Delete Project"


  • The system displays a confirmation message. The user clicks the button "Yes". 

  • The project is deleted and the user could not find it and work with it anymore. The user is transferred back to his/her personal dashboard. 


Delete Project

Users can delete a project for which they have the necessary rights - they are the project creator, the administrator and the project manager. The project is not deleted physically, but its contents is just hidden from the users.

  • The user selects the action from:.
    • Project dashboard/ "Delete Project"

    • View Project Profile/ Actions/ "Delete Project"

    • Personal dashboard/ My projects dashlet/ "Delete Project"


  • The system displays a confirmation message. The user clicks the button "Yes". 

 

Automatic actions after action Delete Project

  • The contents of the projects are deleted (marked as deleted) i.e. the tasks, the cases, the documents (created and uploaded) and the workflows.
  •  The following items from the contents are not marked as deleted: 
    •  domain objects,
    • attached documents or objects from other projects,
    • revisions of documents or objects.
  • The only way to find out which of the contents is not deleted is to perform a search about the project.

Еxample: A project may have "205" in its title and in the title of all its contents, and may contain:

    1. a case with a title "205 Case 1".
    2. an intelligent document with a title "205 iDoc 1".
    3. a revision of the intelligent document with a title "Revision2: 205 iDoc 1".
    4. a document that was uploaded to the intelligent document with a title "205 Uploaded Document 1".
    5. a sub-document to the intelligent document with a title "205 SubDocument 1".
    6. an uploaded document to the sub-document with a title "205 Uploaded Document to Subdocument 1".
    7. attached document from another project (there is "204" in its title) with a title "204 Attached Document 1".
    8. a created domain object with a title "205 Book 1".
    9. a revision of the domain object with a title "Revision 2: 205 Book 1".
    10. an attached domain object from another project (there is "204" in its title) with a title "204 Attached Book 1".
    11. a task within the case with a title "205 Case 1 Task 1".
    12. a workflow within the case with a title "205 Case 1 Workflow 1".

After the project is deleted and a search is performed with the project title i.e. "205", the following primary objects appear in the results:

  1. Revision2: 205 iDoc 1 (revision of a created document)
  2. 205 Book 1 (domain object)
  3. Revision 2: 205 Book 1 (revision of a domain object)

When a new search, with a "204" search term, is performed, the two attached primary objects are present in the results:

  1. 204 Attached Document 1 (attached document from another project)
  2. 204 Attached Book 1 (attached domain object from another project)
