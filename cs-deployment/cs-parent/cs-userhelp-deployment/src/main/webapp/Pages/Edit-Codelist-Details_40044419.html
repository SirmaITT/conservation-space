Sirma Platform : Edit Codelist Details

Only users with administrator permissions may edit codelist data.

  • The user performs a search for codelists and opens a selected codelist in preview (1).

    • The "Edit code list" (2) option is active only on the preview page of the codelist details of a currently valid codelist or of a codelist which validity period has not yet started.

 The user clicks "Edit code list" (2) .

  • The system displays the code list  details in editable form. Formal and business validation of codelist data are applied on the form. 

    • The user may edit the details of the code list ( white fields). Details in another language may be added (1-2) or removed (3). 
  • When the data is entered the user has two options:
    • To save the changes for the codelist  clicking the button "Save" (4).
    • To cancel the changes clicking the button "Cancel" (5) - in this case no data is modified and the form is displayed in preview mode.
      • When the administrator selects to cancel the changes for the codelist, the system displays a confirmation dialog form.
        • If the administrator confirms the operation the system reverts all changes and the codelist details form is displayed in preview mode.
        • If the administrator does not confirm the operation system returns to the codelist edit form with latest changes made.
  • The user selects the option to save the changes in the code list (4).

The following validation rules are applied on the codelist edit form:

  1.  the codelist identifier (ID) cannot be changed (the field is not editable on the form)
  2. the validity start date is required field
  3. the validity start date field is editable on the form
  4. the validity end date is not changed by default - if it was null, it remains null; if it was set, its value is kept and not changed by the system
  5. the validity end date field is editable on the form
  6. the validity end date must be later than the current date and time and is round up to next whole hour (eg. 10:00, 11:00, etc.)
  7. the validity end date must be later than the validity start date
  8.  the validity start date can be round up to whole hour (10:00, 11:00, 21:00, etc.)
  9.  the validity end date can be round up to whole hour (10:00, 11:00, 21:00, etc.)
  10.  the masterCL is a drop down field containing all valid (currently and in the future) codelists for the tenant
  11.  the "Display type" field is a drop down field with options: Value, Description, Value and Description
  12.  the "Sort by" field is a drop down field with options: Value ascending, Value descending, Description ascending, Description descending, Order ascending, Order descending
  13. the description in English is a required field

If there are validation errors on the form, the Save button is not active and data cannot be saved.
