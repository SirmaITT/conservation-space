Sirma Platform : Upload Document as Child of Another Document

The user is able to upload a document as child of another document (both uploaded or created into the system). A relationship of type part of/ has child is created between the two documents.

The action is used when a document has some attachments, like images or another documents which may be uploaded directly in the document.

Rules:

  1. There is action Upload document both for iDocs and uploaded documents into the system.
  2. When the action Upload document is selected, then the Upload document screen is opened and the user could upload one or many documents.
  3. There is no restriction on the types of documents that could be uploaded.
  4. A relationship of type "part of/ has child" is created between the two documents.
  5. The uploaded documents are visible in the relationship widget.
  6. When the parent document is published all its children are also published.
  7. The child document of a document cannot be published alone.

Example:

  • The user selects the action "Upload Document" (1-2) for a document in the official section of a case.

  • The Upload Document picker opens and the user selects a document from his PC , selects the type of the uploaded document and performs the action (1-2-3-4).
    • If needed the user fills in the mandatory data properties for the selected document type of the uploaded document.

  • When the action is completed (1) the user clicks "Close"(2).

  • The user opens  Relationships tab of the document (1-2). The automatically created relation of type "Has child" is displayed in the relationships widget  (3) from the document to the uploaded document.

  • The user opens  Relationships tab of the uploaded document (1-2). The automatically created relation of type "Part of" is displayed in the relationships widget  (3) from the uploaded document to the document.

  • The user opens the official section where the documents are (1-2). For for the uploaded document (2) there is no action "Publish document" (3-4).

  • The user performs the action "Publish document" for the document (1).
  • Both become revision 2.0 (2).
