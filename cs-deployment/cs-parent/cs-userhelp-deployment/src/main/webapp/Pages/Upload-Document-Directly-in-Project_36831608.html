Sirma Platform : Upload Document Directly in Project

The user is able to upload documents directly in the project.

The documents in the project are not stored only in cases, because some organization need to use projects as document stores without having cases in the projects.

Rules:

  1. In the project there is action “Upload document” in the list with all actions on project level.
  2. Only document types valid for the respective project, or for the cases in the project, could be uploaded in the library.
  3. Part of/ has child relationship is created automatically between the uploaded document and the project.
  • The user opens the project dashboard and selects the action “Upload document”  from the list with all actions on project level (1-2-3).

  • The system opens the form for uploading documents (1).
  • The user selects the document for upload from the local PC (2-3-4-5). More then one documents may be selected.

  • The selected document is loaded in the form (1) and the user selects from the drop-down list of allowed document types the document type (2-3). The user may enter a description for the document.
    • If needed the user fills in the mandatory data properties for the selected document type of the uploaded document.

  • The selected document type (1) appears in the corresponding field.
  • The user selects "Upload All" (2).
  • When the process finishes (3) the user closes the form (4).

  • The uploaded document appears in the Content dashlet (1) in the project dashboard.
  • The user opens the document (2).

  • In the Relations section of the document (1) the automatically created relation between the document and the project "Part of" (2) is displayed.
  • The user opens the tab "Relationships" (3).

  • The automatically created relation between the project and the document "Has child" is displayed.

  • The document is visible in the library of the project (1-2).
