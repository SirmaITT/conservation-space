Sirma Platform : Remove Object from Case

Users with relevant permissions are able to remove (detach) objects from case. The removed (detached) objects are not deleted but are only removed from the case.

Rules:

  1. There is "Remove Object" action on the object in the Objects dashlet on the case dashboard or in the Object section of the case.
  2. The action is active only in the context of a case and when a relationship of type Has Attachment/ Is Attached To exists between the case and the object. (Note: one object could be attached to more than one cases, or not to be attached to any case)
  3. The action is active only when the object is explicitly attached to the case, but if it is attached to cases in the project it is not active.
  4. When the action is selected the relationship of typeÂ Has Attachment/ Is Attached To between the project and object is deleted.
  5. Only the users with Consumer role in the project cannot detach objects
  • There is an attached object on case level. Relationship of type Has Attachment/ Is Attached To exists between the case and the object.

  • The user selects the action "Remove Object" for the Objects section of the case (1-2-3).

  • The system displays confirmation message and the user confirms the operation.

  • The object is removed from the case.

  • The relationship of type Has Attachment/ Is Attached To between the case and the object is deleted.

  • The action is recorded in the Audit Log.
