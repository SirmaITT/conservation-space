Sirma Platform : Generate a document based on a template with the completion of a workflow

The user is able to define a template of a document (in the definition), and when a workflow is completed (or in other transition in the workflow), a document based on the respective template to be generated.

The user is able to set up where the generated document to be stored (for example the generated document to be saved as parent of the document processed in workflow, or in a selected project or case).

For example a document of type Review and Approval Log (RAL) to be generated when a workflow is completed. In this document some details of the current workflow  to be displayed: (like start date, description, priority), who and when completed a task in the workflow. The RAL document to be created as a child of the document published in the workflow.

Rules:

  1. The user is able to set up the template of the document that should be generated.
  2. The user is able to set up the transition in the workflow that generates the document.
  3. The user is able to set up the location of the document after its generation.
  4. The document is generated as a child of the created revision of the document or domain object processed in the workflow. If there are more than one documents/ objects processed by the workflow then the document is created as child of the revision of the first document in the list, and is attached to (or related) with the rest.
  5. The user is able to set up if the document to be send via e-mail to some users.
  6. The document is created automatically as a revision and cannot be changed.
  7. The permissions of the document are inherited from the parent document or parent domain object.
  8. The document is attached to the workflow from which is generated

 

Example: A template has been defined for generation of document when the workflow of type WFTYPE992 is completed.
  • The user starts the workflow for the document which was attached in the official section of a case (Test Case 1) in project (Pr16APR) (1-2-3).

  • The user selects the workflow type (1-2).

 

  • The user fills in the necessary data in the starting task and starts the workflow (1-2-3-4-5-6).

 

  • The assignee of the task opens it from User's dashboard/ My Tasks dashlet (1-2).

 

 

  • The assignee comments and selects action "Approve and Publish" (1-2).

 

  • A document is automatically generated (1) after the completion of the workflow.
    • The document cannot be edited (2).
    • Relation of type Part of/ Has child is created between the generated document and the published approved revision of the document processed by the workflow, visible in the Relationships widget (3-4-5).
  • E mail is send to the selected users.  In the e-mail content  there is a link to open the automatically created document.
