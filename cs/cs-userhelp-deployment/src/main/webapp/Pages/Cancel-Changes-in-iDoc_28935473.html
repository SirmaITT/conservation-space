Sirma Platform : Cancel Changes in iDoc

The user is able to cancel the changes which he/she has made in document while the document is still in Edit Mode.

  • To cancel the changes which the user is making in a document while in Edit mode, the user selects the "Cancel" button.

  • The system displays a confirmation message and the user confirms the Cancel operation.

  • When the changes are in a new document (the document is just loaded for content creation but still has not been saved even once and is not in version 1.0), then the document is not created.  A record of this document in the system does not exist.

This is the case when an user selects "Create New Document" action and then one of the existing templates. The template is loaded and the user starts creating content in it.  For some reason the user realizes that this is not the proper template and cancels the creation.

  • When the changes are in an existing document,  then:
    • No changes that have been made in the content are saved.
    • The latest saved version of the document is loaded in Preview mode.
