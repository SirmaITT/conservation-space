Sirma Platform : Edit iDoc

Preconditions

  1. Only an user with respective permissions could edit a document.
  2. Only one user at a time could edit an iDoc.
  • The user selects "Edit" action for the document in Preview mode.

    • The document is locked for editing by other users. There is a superscription on the document visible for all other users "This document is locked by <UserX>"

  • The system opens the last version of the document in Edit mode. While the document is in edit mode the user could manage its content (1-2). He/she is able to add/ remove sections and widgets, to add and format text in the document (for details see iDoc Content Management).

After the user finishes with the changes, he/ she clicks the "Save" (3) button.

  • The Edit mode of the document changes to Preview mode. All the changes made in the document content (widgets, sections, text, formatting) are saved (1). In the document properties section the following fields are updated:
    • Version (2) - Saving changes in a document creates always a new minor version of the document (1.1, 1.2, etc.)
    • Last Modified By(3) - the user who made the latest changes
    • Last Modified on (4) - the date and time the changes in the document have been saved

A new record is created in Version History section of the document (5). The previous version of the document is saved and displayed in the section so any user could always review and revert to a previous version if necessary. For more details on Version section of a document see iDoc Content Management.
