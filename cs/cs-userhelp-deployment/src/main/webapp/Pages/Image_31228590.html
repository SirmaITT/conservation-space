Sirma Platform : Image

  • The user selects Insert Image widget from the Widget Palette and inserts it into the iDoc (1-2).
  • The user enters title (3) and selects "Manage Properties" action (4-5).

  • The object picker screen (Insert Image) opens with following sections:
    - Search - the basic search screen, for search in the system
    - Browse - the ability to browse the project tree
    - Upload - the ability to upload from the local PC

    • In the  "Search" section of the picker the basic search screen is opened, where the user searches for an image in the system. The types of objects allowed for search are restricted only to documents, and the search result display only images (png, tiff, gif, jpg, etc).

    • In the "Browse" section of the picker the project tree is opened, which the user is allowed to browse, but the only objects allowed for selection are the images.

    • In the "Upload" section the user selects image from his/ her PC.

      • The user selects an image to upload (1-2-3-4-5)

      • The image is selected (1) and user clicks the button "Upload Document" (2).

      • When the upload process is completed (1), the user  may remove the uploaded image (2). More images could be selected.
      • The user selects the image (3) and may enter description (4) for it.
      • The user clicks "OK" (5).

  • The image is displayed in the image widget in the iDoc and is saved as an attached document in the case in the same section where the iDoc resides.
    • The user could resize the image to specific width in the document. An image (no matter how big it is) could not exceed the width of an iDoc.
      • If the original image width in pixels is bigger than the iDoc width, then in the Image width field is displayed the width of the iDoc by default.
      • If the original image width in pixels is smaller than the iDoc width then in the Image width field is displayed the image width by default.

In the widget there is a field "Image Width" (1) in which the user could enter the size of the image in pixels, that he/ she wants to be displayed in the document. The user could enter smaller width than the iDoc width or then the original image width.

In the widget there is an action "Annotate" (2). The action is active only for inserted images in the widget, which are located into the system (cannot annotate an image inserted from the web)

  • By default the image is displayed in the widget without annotated areas on it, and the user could go into Image Annotation tool and select (search/ filter) those that he/she wants to be displayed.
  • The user selects the action "Annotate" and the image is opened in the Image Annotation (IA) tool.

The IA tool works with all the functionality it has.

    • The user could create new annotated areas (1-2-3) on the image or could filter the existing annotated areas. The user could zoom in/ zoom out the image, etc.
    • After the user completes the annotations he/she clicks "Save" (4) and exits the IA tool.

  • When the user exits the IA  and returns back to the widget he/ she saves the iDoc.

    •  The annotated areas, which are result of the sort/ filter are displayed in the image in the widget. Only the annotated areas, not the comments associated with them are displayed.
    • The annotated areas displayed in the widget are a snapshot of the existing annotations.
    • If there are new/ deleted/ edited annotated areas on an image, then they are not displayed automatically on the image. Only the user could open the IA tool from the widget and select to update them.

 

  • If the image is zoomed in, then only the area selected for display for the user (look in the yellow area on the right) is displayed in the widget.
