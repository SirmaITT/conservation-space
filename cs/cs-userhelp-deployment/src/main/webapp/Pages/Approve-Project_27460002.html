Sirma Platform : Approve Project

When the project is created in the system and the initial project environment is already set up - for example the project schedule is created, there are users assigned, there is a decision on the project start and end dates, on the budget, etc, then the project manager could ask for approval from higher level authority on the project parameters. The project manager could approve the project by himself also or if there is an approval based on a document to select the name of the user who made the approval. 

The approval is not a mandatory prerequisite for starting a project (see Start Project), a project could be started without being approved. This depends on the project manager choice. 

The use case describes only the requirements for approving a project by performing the action "Approve", but a project could be approved also with a special review and approval workflow, which if completed successfully could automatically change the status of the project to "Approved".   

Preconditions

  1. The action "Approve" for a project could be performed only for a project in state "Submitted".  In any other state of a project, the action is invisible.
  2. A project could be approved only by an user with the role of "Project Manager".  
  •  The user may start the action from:
    • Project dashboard/ Actions/  "Approve Project" (1-2)
    • View Project Profile/ Actions/ "Approve Project"
    • Personal dashboard/ My projects dashlet/ "Approve Project" for a project in status "Submitted".


  • The system opens the project details in edit mode. The user enter a comment and saves it. (1-2-4-5-6).

When the pop-up screen for entering a comment opens, the name of the current user is automatically entered (3) and cannot be edited.

 

  • The entered comment is saved (1). The user clicks the button "Approve Project" (2) at the bottom of the page.
    • (Not Ready) The approval process may be performed by a special workflow and the user may start it in this step.

  • The system opens the Project details screen. The project status is changed to "Approved" (1) on any of the following conditions:
    •  The approve action is successfully completed. 
    •  (Not Ready) An approval workflow successfully completes in the "Approved" status. 

The user is able to perform on the project the actions shown in the list (2-3).

  • The entered comment is visualized on the Personal dashboard of the current user (1-2).
