Sirma Platform : Browse Case

A case in any state may be opened for preview by any user with access to that case.

When a case is opened for preview, an user with corresponding permissions may add/ delete documents, objects, start workflows or initiate other actions depending on the state in which the case is.

  • The user opens a created case for preview from:
    • Result from search cases/ select link to a case -screen "Case Dashboard" is opened
    • Case details - related cases/ select link to a case -screen "Case Dashboard" is openedÂ 
    • Result from filter cases/ select link to a case -screen "Case Dashboard" is opened
  • When an attached document is selected, it is opened in preview.
  • When the case is opened in preview and the user who opened the case has corresponding permissions, according the state in which the case is, he/ she may perform actions like: attachment of new documents to the case, move documents from one section of the case in another, start workflow, add/ delete related cases , manage objects (1). More actions may be performed selecting them from the drop-down menu "Actions" (2-3), or selecting actions from the dashlets (4).
  • When any action from the mentioned actions above is performed, the following attributes of the case are updated:
    • Modified on - the date and time of the last update
    • Last modified by - the user who modified the case last

  • (Not Ready) Information about all actions carried out with the corresponding case are preserved in section "Case History": who and when has carried out these actions, in which state was the case then.
