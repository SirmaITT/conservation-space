Sirma Platform : Edit Project

Preconditions

1. Only the Creator of the project, or an user with role of a Project Manager for the project could edit project details.
2. The Project Details can not be edited if the project is in a final state: "Stopped" or "Completed".

  • The user starts the action from :
    • Personal dashboard/ My Projects dashlet/ Actions for chosen project/ Edit Project
    • Project dashboard/ Project details dashlet/ Actions/ Edit Project (1-2)
    • Project profile view/ Actions/ Edit Project

  • The system opens the "Project profile" screen in edit mode. It depends on the project type which fields are editable and which are not. There could be a project type, defined with zero editable fields, or other with most of the fields editable.
    • No other Actions for the project are permitted.
    • The Unique Identifier ID , which is system generated and the Status of the project cannot be edited (1).
    • The user edits the necessary details of the project (2-3-4)
    • When ready with the updates the user clicks the button "Save" (5). 

  • The system saves the changes and opens the Project Profile screen:
    • the entered changes are saved in the corresponding fields (1-2-3)
    • automatically is updated the information for:
      • Last modified by - the user who edited the project details
      • Last modified on -the date when the project details have been edited
    • the "Actions" list displays the available operations on the project for the current user (4)

  • The saved changes in the project's details are visible also in the Project details dashlet on the Project dashboard.

  • (Not Ready) All changes may be tracked in the audit log.
