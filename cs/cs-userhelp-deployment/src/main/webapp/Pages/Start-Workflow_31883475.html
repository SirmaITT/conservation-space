Sirma Platform : Start Workflow

Unlimited number of workflows may be started for a case.

Preconditions

Workflows can be started only for cases in an active state.

  • The user starts the operation from:
    • Case Dashboard/  Actions/ Start Workflow 
    • Case Dashboard/ Case details dashlet/ Actions/ Start Workflow
    • Case Dashboard/ Workflows dashlet/ Actions/ Start Workflow
    • Case details page/ Case details/ Actions/ Start Workflow (1-2)

  • The system opens a screen for selecting a type of workflow (from a drop down list), that may be started for the corresponding type of case . The user selects the type of workflow (1-2)

  • The system opens an empty form of the selected workflow type (1) where the user:
    • enters or changes data in all mandatory fields (2 and 5) of the selected type of workflow
    • may enter data in non mandatory fields (3,4,5)
    •  may relate other objects to the workflow  which will appear in the first task of the workflow (6)
  •  The user clicks the button "Start Workflow"(7).

  • The system opens the page from where the action was started (here - Case details (1)). The user may review the workflow/ s clicking tab "Workflows" (2) for the case.

  • The system displays the list of workflows in this case.
  • The user selects the link to open a workflow.

  • The system opens the workflow details page where the user may:
    • review: Workflow Diagram, Workflow details, Current and Completed Tasks, Related objects (case, tasks, documents etc.), if this section is requested by the customer.
    • review or enter Comments on the workflow
    • perform Actions available in the drop-down list (1-2)

  • The user opens the Case dashboard.
    • In the Workflows dashlet there is information about the started workflow and the available actions on it for the current user (1-2).
      • For every workflow there is information about: Name of workflow, State of workflow, Started by - the user who started the workflow, Started on - date and time (seen in the tool tip)
      • Available filters for workflows are: Not Complete (set by default), All Workflows, High Priority, Overdue
    • In the Tasks dashlet information about the active task in the workflow (3) and the available actions (4) which the current user may perform.
    • In the Documents dashlet the added documents in the workflow/ case would be displayed.
    • In the Colleagues dashlet the assignee of the first task of the workflow which was started in the case appears with the role of "Collaborator" (5). This role of the assignee would be preserved until he/ she completes or reassigns the task.

Information is stored for all started workflows for each case, no matter active or not.
