Sirma Platform : Review Workflow

The user is able to review existing workflows in the system.

Preconditions
The user may review the workflows only for cases to which he/ she has access.
There is at least one workflow (no matter of its state) for the selected case.

  • The user starts the review of the workflow from:
    • Personal Dashboard/ Workflows dashlet / Name of workflow (link) (1-2)
    • Project Dashboard/ Workflows dashlet / Name of workflow (link) 
    • Case Dashboard/Workflows dashlet / Name of workflow (link)
    • Case Dashboard/ Workflows section/ Name of workflow (link)

  • The system opens the workflow details screen. For each workflow the following sections are displayed:
    • Workflow Diagram(1) - a state diagram of the process, with red is the current  stage in which is the process.
    • Workflow details(2)  - Name and Status of the workflow (at the top of the screen), Workflow description, Workflow status, Priority, Due date, Started by, Started on, Message
    • Current Tasks(3) - a list with all tasks, started in the workflow which are not in a final state.
    • Completed Tasks (4) - a list of all completed tasks in the workflow. (i.e.that are in a final state)
    • Related Objects - a list with all relationships of the workflow with other primary objects (project, case, document, task, etc). The section is displayed if ordered by the customer.
    • Comments (5) - all comments and replies on the workflow (6).
      Actions (7) - a list of available actions for the current user. If the workflow is "In Progress" the originator of the workflow has the possibility to stop the workflow by selecting the action "Stop Workflow" (8).

For each task are visualized: unique ID, Task Type (name), Due Date for the completion of the workflow, Status of the task, Assignee, Message. The Task Type is a link to open the task for review/ edit.
