Sirma Platform : Manage Project Resources and Roles

  Preconditions  

  1.  Only users with the role of "Project Manager" on the project, could manage project resources. This means they may: 
    1. Assign new users/ groups to the project
    2. Remove assigned members on the project
    3. Change user's roles on the project 
  2. There always must be at least one project member with the role of a "Project Manager". If there is only one user left with this role, then the role could not be changed to another one.  
  3.  Project resources cannot be managed for projects in the final states "Stopped" or "Completed".
  •  The user starts the operation from:
    •  Project dashboard/ tab Project Members (1) or
    • Project dashboard/ Project details dashlet/ Actions/ Project Members(1-2-3) or
    • View Project Profile/ Actions/ Manage Resources

  • A screen opens, where the user could search for users or user groups, assign/ remove users or user groups from the project and change permissions of users or user groups.
    • The screen is divided in two sections - Add Users/ Group(1) and Project Members (2)
    • Both sections are paginated (3).
    • There is a search field in both sections (4) where the user could search by typing user/ group name or part of it and press "Enter" key on his/ her keyboard.
      • Section Add Users/ Group (1) - here the project manager searches for users or group of users (4-5) to assign them to the project. The list may be sorted by name ascending or descending (6).
        • The user could select from one of the values  - Add users or Add groups (5).  By default is selected the value "Add users" and when the screen is opened, automatically are listed all existing users, sorted alphabetically by first name.  
        • When "Add users" (5) option is selected, then the users in the section could be sorted by the following criteria "By first name", "By family name", "By user name". The sorting could be done alphabetically - ascending or descending.
        • If the user selects "Add group" value -in the section are listed all of the existing groups of users. 
        • When "Add groups" option is selected then the groups in the section could be sorted only ascending or descending by group name.
        • Next to each user/ group, who is not already a member of the project, there is an add button (7).
      • Section Project Members (2) - here are listed all current project members, with their roles. 
        • Next to each user/ group is displayed the role (8) assigned to that user/ group in the project
        • There is a remove button (9) next to each role. If the remove button is selected then that user/ group is removed from the project.
  • The Project Manager adds a selected user to the project clicking the correspondent "+" (7) button.


  • The selected user is transferred into section "Project Members" (1) and the add button for that user is hidden. The user/ group is added into the project with an empty - there is no role is assigned yet. 
  • From the drop down menu the project manager selects the role which to be assigned to the selected user/ group (2-3). The roles that could be selected are: Project Manager, Collaborator, Contributor, Consumer  


  • When ready with the changes the user clicks the button "Save" (1-2).

If the user does not want to save the changes that he/ she has made on the project resources, he/she could select the "Cancel" button or leave the screen without saving changes. 

  • The changes in the Resources and Roles of the project are visible on the Project dashboard/ Colleagues dashlet (1-2-3).

  •  (Not Ready) If the user has subscribed for receiving notification for new/ changed assignments on projects, then when he/she is added or removed as a member of a project (as single user or group of users) , he/she is notified via e-mail for this. See "Manage Personal Profile".
  • No matter if an user is notified via e-mail for his/her assignment to a project or not, the project he/she is assigned to appears in his/her personal dashboard, "My projects" dashlet.
  • When a task is assigned to an user who is not a project member, the user automatically is added into the list with project members with the role of Consumer. The project manager could after that change the role or delete the user from the list after the task is completed.
