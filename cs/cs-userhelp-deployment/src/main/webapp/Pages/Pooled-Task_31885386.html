Sirma Platform : Pooled Task

Several users or a group of users are selected as task assignees, but actually the task remains "Unclaimed" until someone from the group claims it.

  • Create Pooled Task
    • Claim Task
    • Release Task
 

Create Pooled Task

  • An empty form for creation of a selected task type (1) is opened.
    • For this type of task several task assignees may be selected. The user enters the necessary details of the task and clicks "Select users" (2).

  • The system opens the screen for selecting users (1).
    • The creator of the task selects assignees from the list (2).
    • The selected users are added into the right section (3).
    • The user clicks "Save" (4).

  • The selected assignees are added in the "Assigned to:" field (1). The user clicks "Create Task" (2).

  • The task is created as an "Unclaimed" (1) task and is sent to all selected assignees (2).

Claim Task

Unclaimed task are displayed in "My Tasks" dashlet/ filter "Unclaimed pooled tasks".

  • An user from the selected assignees to whom the task has been assigned opens the unclaimed task (1-2).


  •  For each pooled task while the still unclaimed there is an action "Claim Task" (1-2) .

The user selects the action. This means that he/she confirms that he/she would be the one from the group of selected assignees that will do the job . 

  • The task is opened in edit mode and:  
    • The task becomes assigned to the user who claimed it (1).
    • The task is removed from the personal dashboard of all other selected assignees.
    • Action "Release Task" becomes active for the claimed task and appears in the task actions menu (2).

The user may edit the task details and complete the task (3) or select another action from the drop down list of actions.

Release Task
  • The user who claimed the task could release it back to the pool (selecting the action "Release task") at any time before finally completing it, which means that he/she rejects to do the job:   
    • The task becomes "Unclaimed" again.
    • The task is displayed on the personal dashboards of all users from the selected assignees, that was initially selected for the pooled task and the action "Claim Task" becomes active.
    • Any user from the selected assignees could claim it again.
