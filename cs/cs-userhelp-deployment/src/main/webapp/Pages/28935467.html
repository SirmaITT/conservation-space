Sirma Platform : Intelligent Document (iDoc) Management

In this section the requirements for managing the content of an intelligent document (iDoc) are described.

The iDoc is treated in the system as a primary object of type "Document". The difference is that the iDoc is with a HTML content.

The iDoc inherits all actions on a Document in the system - Clone, Move, Delete, Edit Properties, Version tracking, Revert to previous version, etc.

Here are described only the actions specific to the iDoc content - creating new iDoc, spell check and grammar check of the content, export in different formats, editing the content.

Each iDoc has three vertical sections: Document tree view, Document content and Comments section.

  1. Tree view section - shows the hierarchical structure of a document.
  2. Content section - shows all the widgets used in the document plus the HTML content (for more details see below).
  3. Comments sectionÂ  - shows all comments on the document. Each document section could be commented.

For iDoc we introduce the term Widget. The Widget is a pluggable and customizable application fragment representing an object or a piece of object. It could represent a whole range of things from simple rich text block to complex semantic queries.
In the iDoc content section the following types of widgets are used: default widgets (which the user is not able to alter) and selectable widgets which the user may select during the creation/ editing of the document. The widgets are used to create the document content and format it in a way that fits best to the user's needs.

When a new document is created based on a template, then all predefined widgets in the template are mandatory.

When a new document is created from scratch then:

1. The Content section consists of the following default widgets:

  • Document Identification - shows the Document ID, Document Title, Bread Crumb navigation
  • Document actions - this widget presents a list of all actions on the document content - Save, Preview, etc. These actions are different from the actions on the object Document - Delete, Clone etc.
  • Text Editor - this is a built in text editor that allows the user to perform basic text formatting actions - headings, font changes, color text, bold text, etc, the use would also be able to insert/ delete/ edit simple tables, insert links to other objects.
  • Documents metadata - displays the document metadata

2. The user could select from the following widgets to insert in a document. One and the same widget could be inserted more than once.

  • Main Object Selection - a widget in which the user is able to select a first class system object and then choose which metadata of the object to be visible in the document. (TBD: How would be decided/selected automatically the object instance to be displayed, a parameter is needed.)
  • Add entity - the user is able to select an instance to be displayed.
  • Add facts - the user is able to add new facts on the selected instance.
  • Actions - for the selected instance, based on the object type the user would be able to perform basic actions.
  • Table of Content - the widget will automatically create document table of content based on the existing headings.
  • iDoc Data Table - the widget is described in a separated section "iDoc Data Tables".
