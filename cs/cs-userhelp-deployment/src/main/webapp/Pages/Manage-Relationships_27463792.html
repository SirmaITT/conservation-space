Sirma Platform : Manage Relationships

The references that could be created between objects are a predefined list, and the user when creating a new reference has to select among the existing in the list.
Each reference has a primary and a secondary label which depends on the object from which it is viewed.

Constraints: The action Manage References depends on the role of the user in the project:

  1. An user with the role of "Collaborator" in a project could manage references to all  objects in the project.
  2. An user with the role of "Contributor" in a project could manage references only to objects created by himself.
  3. An user with the role of "Consumer" could not manage references, but only to view the existing references.

References between objects could be managed at any state of the object, either active or inactive.

  • The user could manage relationships from each primary object landing page (1), section "Relationships" (2).
  • A screen opens where the user could view details about all currently existing relationships from/to the current object.

The existing relationships are described in a table format (3) with the following columns:

      • Relation type (4) - showing the label of the reference created between the two objects.
      • Related object (5) - information for the objects with which (from/to) a the relationship is created.

From this screen, the user could also create new relationships (6), edit an existing one (clicking on its relation type) and delete a selected relationship (7).

 

  • By default in Manage Relationships screen are displayed all existing relationships (user created and automatically created) except the parent-child relationships of the object.
    Example: A project has many cases in it, many tasks and workflows - the project is in "has child" relationship with all these objects. If we show all these relationships, then the user could get lost and skip other important relationships.
    The user is able to explicitly select to view the parent/child relationships.
