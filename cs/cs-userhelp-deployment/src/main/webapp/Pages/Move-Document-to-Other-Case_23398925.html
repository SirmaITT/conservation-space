Sirma Platform : Move Document to Other Case

The action is performed when the user wants to move a document from one case in another case.

Preconditions

Documents may be moved only out of a case in an active state.
Documents may be moved only in a case in an active state.

  • The user starts the action from:
    • Personal Dashboard/ My Documents dashlet/ Actions/ Move Document to other case for a selected document
    • Case Dashboard/ Documents dashlet/ Actions/ Move Document to other case for a selected document
    • page Case Details/ tab Documents/ Actions/ Move Document to other case for a selected document (1-2-3)
    • Document preview/ Actions/ Move Document to other case

  • The system opens a screen for selecting case.
    • The user is able to search for cases using the criteria described in "Search Case" (1-2-3-4)
    • When ready, the user clicks "Search" (5).
    • The system displays the results. The user chooses the target case and selects the section (6) in which the document should be moved.
      • If the target case has more then one sections, all sections of the case are displayed.
    • The user clicks the button "Move Document" (7).

  • The document is moved in the selected section of the target case. (1-2-3). The document is moved along with all its details in the new case.

  • When the operation is finished the document is not visible in the source case (1-2).
