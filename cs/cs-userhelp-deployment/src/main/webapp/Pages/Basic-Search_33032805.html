Basic Search

  • The user selects the search from the main menu (the menu with the link to the Personal Dashboard, Log out/ etc. ) and the basic search screen is opened.

  • The basic search form opens:
    • "Search" and "Reset" options (1) are on top of the filters (2).
    • The following filters (2) are implemented in the basic search form: Object type, Object relationship, Search context, Date Created From, Date Created To, Created By.
    • A message is displayed informing the user of the number of objects found (3). The results are displayed in a list (4) which is paginated.
    • The results may be sorted by: Title, Type, Modified On, Modified By, Created On, Created By. The default sort option is Title (5). Other potions may be selected from the corresponding drop-down list (5).
    • The results may be displayed in ascending or descending order (6).
    • In the basic search form the "Full Text and Keyword" search option (7) may be used in combination with the basic search.
    • There is a radio button switching to the Advanced search option (8).
    • The user is able to create and load previously created and saved searches (9).

  • In most search criteria fields an auto suggest is implemented and the user is able to select from list the necessary item (1-2).

  • The user may select in some filters (Object type, Object relationship, Search context, Created By)Â  more than one item (1). Each selected item may be deselected by clicking its button "x" (2).

  • Dates may be selected from the built-in calendar.

  • In the "Search context" field the user is able to select any first class system object as context for the search. The system automatically displays the titles of the 25 most popular (used) objects in the system (if not possible - the 25 recently modified (with the most recent date of creation/ modification). The list with auto suggested objects is reduced with every next letter that the user enters in the Search Context.
  • When the search criteria are entered (1-2-3), the user initiates Search (4) and the results are displayed (5-6).


Automatically select search location - context

  • When the user select Search, while working on a project (1-2).



  • The system opens the Basic Search screen and the field Search context automatically is set to the project the user is working on.
