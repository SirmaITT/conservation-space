Sirma Platform : Create New Project

The main characteristic of a new project is the Project Type. Project types are predefined into the system and aim to allow different users to be able to use data and features specific to their environment, department, job.

For a Project Type could be specified:
1. Specific project details - these are the details that we need to keep into the project profile, and which characterize it better. For example for some project types we may need to keep budget information, for others we may want to keep project objectives, for third - information about partner institutions, etc. Also we may want project IDs for different types to follow different templates (some to start with PRJ+counter, others to start with DEV+counter+date, etc. ) Project types allow users to be flexible and not to stick to one set of data.
2. Specific types of sub-projects/ cases per project - a project type may require that only certain types of activities could be performed, so an user when creating new sub-projects/ cases will be able to select them from a restricted list.
3. Specific workflows for a project type.  A list with available workflows is set up. For example one project type may require a workflow for review and approval before being started, other may require a specific workflow for final completion, etc.

The project of type "Generic" allows users to create projects with no restrictions on workflows and sub-projects.

  • The user is able to create a new project from:
    • Personal dashboard/ main menu Projects/ New Project (1-2) or
    • Personal dashboard/ "My projects" dashlet,/ New Project (3-4)

  • The system opens the screen where the user selects the project type from a drop down menu (1-2).

  • The system opens the screen for the entering the project details. The details are grouped in sections (1-2-3-4-5-6):
    •  The user should fill in data in all mandatory fields and may enter information in non mandatory fields.

Main Project Details: Project type, Title, Visibility, Priority

      • The user selects the project visibility (3) into the system, either visible only for the assigned project members or to all users. By default the value is set to "Visible for all users" - in this case every user who is not a member of the project and all project members will be able to browse it. The user could change the visibility to "Visible for project members only".
    • The user selects the button "Save"(7).

The button "Save" in new project details screen is active, only when the mandatory fields for saving a project are filled.

The button "Cancel" (next to button "Save") is active at any time, and the user could cancel new project creation at any time. If "Cancel" is selected after updates are made, the user is warned that changes will be lost and will be given a chance to abort the cancel operation.

  • The user is transferred automatically to the "Project profile" view of the newly created project.
    • When new project is created and its details are saved for the first time the following project metadata are generated automatically:
      • Status (1)- the project status of a newly created project is automatically set to "Submitted"
      • Unique Identifier(ID) (2) - the unique project identifier is given automatically to the project. For each project type could be set up different convention for ID generation.
      • Created on - the date and time stamp of when physically the project record is created.
      • Created by  - in this field is kept the name of the user who created the project, this user could be different from the project manager.
      • Comments (3) section - where comments and replies on comments on the project are saved. When a new project is created the section is empty.
    • There is a Project members (4) section - displaying the list of current project members and their role in the project. When a new project is created the Creator of the project is assigned automatically as a resource to the project with the Role "Project Manager". Later on he/ she could change his/ her permissions (see Manage Project Resources and Permissions).
    • There is a Actions (5-6) -showing the available actions, which may be performed by the current user.

In this view the user could continue with entering additional data to the project profile, specific for the project type, assigning users to the project, setting up project events, creating new tasks and sub-projects (cases), etc.

  • The created project is visible on the Personal dashboard of the user. The user is able to:
    • Open the Project dashboard, clicking on the link (1).
    • Perform different operations on the project (2-3).

  • On the Project dashboard the user is able to review all information for the project.
