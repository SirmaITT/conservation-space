Sirma Platform : Save iDoc as a Public Template

The user is able to create a public template from an existing iDoc.

Two templates with the same Name could not exist in the Public templates list.

  • The user opens the document, which he/ she wants to save as a template and selects the action "Save as template"(1-2).

In the example below the document is of type "Empty Document" and with name "Test Report 1" and with one widget "Available Checklists" and one section "Document Details".

  • The system opens a screen, where the user:
    • enters the name of the new template (1)
    • selects whether the new template to be primary for that type of document (the check box must be selected) or not (the check box must be empty) (2)

Each document type could have more than one templates associated to it:

- one primary template - the primary template is loaded when the document is selected for creation

- alternative templates - other templates that could be used for creation of the same document type

    • clicks the button "OK" (3)

The new template will be created for the Document Type  "Empty Document", same as the type of the document from which the action "Save as template" was started.


  • The system saves only a template of the selected document, which means:
    • Saved are all existing content sections and sub-sections (the structure of the document) with their names and the text in the sections.
    • Saved are all existing widgets in the sections - but with no information for specific object instances or additional text in them. The object information will be extracted automatically with the instance creation.
  • The user creates a new document of type "Empty Document".  The system opens the primary template of the selected type of document. 

  • The user may change the primary template of the document, selecting a new template from the drop down list  of available templates associated with the current document type(1-2).



  • The system loads the selected template.

 

 

  • The user makes the necessary changes (1-2) and saves the new document (3).

  • The new document of type "Empty Document" is saved for the first time (version 1.0) with the new title (1-2-3).
