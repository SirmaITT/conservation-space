Workflows

The workflow is a sequence of tasks to be fulfilled in order to achieve a particular result. Each task can be performed by a person, a group, or automatically.

Main workflow metadata attributes (details)
  • Type of workflow - (example Unlimited Task Assignment)
  • Description - a brief description of the purpose of the workflow
  • Created by - the name of the user who started the workflow
  • Start Date - date and time when the workflow was started
  • State - the current state of the workflow according its state diagram 
  • Message - text message to the assignees of the tasks in the workflow
  • Due Date - the planned date for workflow completion
  • Priority - the priority of the workflow (low/ medium/ high)
  • Case - link to the case, for which the workflow is started
  • Completion Date - the date when the workflow is completed
Main workflow states
  • In Progress - an active state of the workflow. A workflow goes into this state after the first task in the workflow is assigned for execution and remains in this state until the final and successful conclusion of the last step of the process or its premature termination.
  • Stopped - a final state of the workflow. A workflow goes into this state either by direct request for termination of the user who started the process or as a result of execution of a task on the process that lead to its termination.
  • Completed - a final state of the workflow. A workflow goes into this state after the successful completion of the last task on the process.
Tasks

Tasks are a mechanism for assigning a particular action to achieve a goal in the performance of business processes.

Main task metadata attributes (details)
  • Type - type of task (example "Task for Execution")
  • Name - detailed description of the task according the type of the process on which the task is executed
  • State - state of task
  • Assignee - the user who executes the task
  • Creation Date of the task
  • Completion Date of the task
  • Comments on the task
  • Outcome of the task
    The following metadata from the workflow are inherited by each task in the process:
  • Due Date - the planned date for workflow completion
  • Priority - the priority of the workflow
  • Case - link to the case, for which the workflow is started
Main task states
  • Not Yet Started - an active state of a task. Each task is in this state when it is created. The condition reflects the fact that the assignee has not yet started working.
  • In Progress - an active state of a task. Task enters this state automatically when it first opened for review by the user. In case the user does not intend to start work on it, he/ she can manually change the task status to "On Hold".
  • On Hold - an active state of a task. The user can manually put the task in this state when he/ she wants to note that he/ she  will start work on the task later.
  • Completed - a final state of a task. Task enters this state automatically when the user performs certain actions and the final goal for the task is created.
  • Stopped - a final state of a task. Task enters this state only upon termination of the workflow, and if it has not been in the final state "Completed".
