Sirma Platform : Clone iDoc

iDocs may only be cloned as copies of their original type. (i.e.an iDoc of type 'Treatment Report' can only be cloned as another object of type 'Treatment Report')

This is because the templates defining the metadata and the content structure of an iDoc make cloning from one type to another type impractical.

  • The user opens an existing iDoc and selects the action "Clone" from the drop down menu of available actions (1-2-3):

  • The system opens the screen for selecting the target section to which the iDoc will be cloned with an automatically assigned unique ID. The user selects the location (1-2-3-4).

  • The System opens the cloned iDoc in the selected location (1). The user edits its content and/ or metadata attributes. After the edit is completed (2) the user clicks the "Save" button (3).

When the user attempts to save the cloned record, the system prompts the user if fields requiring unique data are flagged.

  • The system saves the changes in the cloned document (1-2-3).
