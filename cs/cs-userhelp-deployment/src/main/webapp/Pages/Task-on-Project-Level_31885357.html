Sirma Platform : Task on Project Level

Preconditions 

  1. Only the task assignee and users with the role of "Manager" on the project could create sub-tasks of a task. 
  2. Sub-task could not be created in a task in status "Completed". 

Task on Project level

  • To create a task on Project level (not related with a case) the user selects from Project Dashboard/ Project Tasks dashlet/ Actions / Create Task (1-2-3-4).

    • The system opens the screen where the user selects the task type from a drop down menu (1-2). The list of all available task types is based on the object definition.  

  • The system opens an empty form for the selected task type (1). The user fills in the necessary data (2-3-4-5-6-7). Fields marked with "*" are mandatory.
  • When the user is ready he/she clicks the button "Create Task" (8).

  • The system opens the task details page of the newly created task.
    • The task is created in status "Approved" (1)
    • The details of the task are displayed (2)
    •  The newly created task is in relationship "Parent/ Child" with the project from which it is created (3).
  • The user clicks "Project dashboard" (4).

  • The created task appears in the Project Tasks dashlet (1).
    •  The integrated pooled task in the Project  is created in status "Approved". 
    •  The pooled tasks are unassigned until they are claimed by any of the users that were selected during the creation of the task as assignees.(instead of the assignee name there is text 'Unassigned')  and appear in:
      • Personal dashboard/ Tasks dashlet - for each user who is selected in the pool of possible assignees.
      • Project dashboard/ Projects Task dashlet - for the project under which the task is started
      • Case Dashboard/ Task dashlet - for the case in which the task is started
    • There is a filter "Unassigned tasks" in the: Task Dashlet in Personal dashboard, Task Dashlet in Project Dashboard,  Task Dahslet in Case Dashboard,  In the Faceted search/ Tasks/ Unassigned task 
      The filter displays all the task which do not have Assignee, and are flagged as "Unassigned". 
    • No actions are available on "Unassigned" tasks until the task is opened and claimed by any of the assignees (2)
  • The user clicks "Project Schedule" (3).

  • The task is displayed in the project schedule as an entry of type "Task" (stand alone task in the project) (1-2-3). The selected assignees for the task are added in the project members team (4) (if not already there).


NOTE: When the project is in status "In Progress" the integrated single assignee standalone tasks are created in status "In Progress" and in the Project Tasks daslet there are actions available for the current user for this task.
