Sirma Platform : Task as Sub-Task of Another Task

Preconditions 

  1. Only the task assignee and users with the role of "Manager" on the project could create sub-tasks of a task. 
  2. Sub-task could not be created in a task in status "Completed". 

Task as a Sub-task of another Task

  • The user selects from Task landing page/Actions/ Create Sub-task (1-2).

  • The user fills in the necessary data (1-2-3) and clicks "Create Sub-task" (4).


  •  The newly created task is  added in the "Sub-tasks" section of the task from where the operation was started.
  • The newly created task is in relationship "Parent/ Child" with the task from which it is created. 


  •  The task is displayed in the project schedule as an entry of type "Task" (sub-task of the entry of type "Task").
