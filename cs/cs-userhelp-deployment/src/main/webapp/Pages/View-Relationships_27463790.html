Sirma Platform : View Relationships

In each object landing page there is a section "Relationships", where the user could view all existing external references from the current object to other objects or from other objects to the current one.

  • The user selects the section "Relationships" (1-2) in an object  landing page (here - Project dashboard).

  • The system opens the screen with the list with related content of that object (here -project). The list has two sections - Relation type (1) and Related object (2).

Each object in the list is represented with its icon, ID, Title and Type. The ID, Title and Type (2) of each object are a link to the respective object landing page (here - case).

The link to the object is not accessible if the user who reviews the current object does not have permissions to the referenced one.

Rules:

  1. If an object is moved to other location, its references remain unchanged.
  2. If a new version of an object is created, the references always remain to the latest version of the object.
  3. If an object content is cloned, the existing references to the object are not cloned.
